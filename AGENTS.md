# DemoNet

策展化的"试玩派对" — 多品类聚合平台（游戏/电影/动漫/桌游/模型/书籍/音乐/数码/咖啡/线下体验）。

## 技术栈

- **前端**：Vue 3 + Vite + Pinia + Vue Router 4 + Tailwind CSS 4
- **后端**：Spring Boot 4.0.6 + JDK 17 + MyBatis-Plus 3.5.16
- **数据库**：MySQL 8.0 / MariaDB 11（生产）
- **缓存**：Redis 7
- **消息队列**：RabbitMQ 3.12
- **认证**：Spring Security + JWT（jjwt 0.12.6）

## 文档维护原则（适用于所有 AI）

1. **以事实为准**：代码、配置、测试结果和已验证的生产状态优先于文档；发现冲突时，修正文档并明确说明验证范围，不能用旧文档覆盖事实。
2. **一个主题一个主文档**：项目概述、结构、数据模型、API、历程、品类、运维测试、问题计划、外部集成、代码质量、CI/CD 分别有唯一主入口；新内容先更新对应入口，不新建同主题的平行文档。
3. **合并而非丢失**：合并文档时保留原文的背景、方案、验收标准、假设与决策理由；将过往内容移入“历史/附录”，并标记“已完成、进行中、待验证、已废弃”。仅可删除与项目无关、重复且无独立细节，或已证伪的内容。
4. **历史与现状分层**：历史计划不等于当前待办。历史内容必须标注日期和状态说明；当前操作步骤、环境变量、端点和命令必须放在主文档前部并保持可执行。
5. **变更同步**：改动 API、数据表、环境变量、运行方式、测试门禁、部署或安全边界时，必须检查并更新 `document/README.md`、对应主题文档和本文件中的受影响说明。
6. **链接与编号一致**：删除、合并或改名文档后，必须更新所有 Markdown 链接、索引、交叉引用和目录表；不得留下指向不存在文件的链接。
7. **可验证表述**：不得把计划写成已实现、把本地验证写成生产验收，或编造测试结果。记录命令时注明前置条件；禁止在文档、日志或示例中写入真实密钥、Token 或密码。
8. **最小必要维护**：不因格式偏好重写无关内容；避免重复技术栈、环境变量和操作手册。可用链接指向唯一事实来源时，不复制容易漂移的细节。
9. **完成前检查**：至少运行 `rg` 检查已删除文档的引用、检查文档索引和链接目标，并执行 `git diff --check`。无法验证的项目须明确记录原因和后续验收条件。

## 快速启动

```bash
bash scripts/setup-env.sh     # 首次：生成 .env
bash scripts/start-dev.sh     # 启动全栈
# 或手动：
docker compose up -d
cd backend && mvn clean spring-boot:run   # :8080
cd frontend && npm run dev                # :5173
```

## Git Commit 规范

```
<type>(<scope>): <中文简短描述>
```

- type: feat / fix / docs / refactor / style / chore
- scope（可选）: frontend / backend / admin / auth / docker 等
- 描述用中文，一条 commit 只做一件事
