# ☕ DemoNet 咖啡板块规划方案

> 项目：**Demonet** — 让各种实体/虚拟事物都能被用户充分了解  
> 类比：影视看预告片 · 音乐可试听 · 游戏下 Demo → 咖啡怎么"体验"？

---

## 一、核心挑战

咖啡的味觉和嗅觉**无法直接数字化**。  
解决方案：**信息密度 + 感官引导 + 社区共创**，让用户在没有喝到之前，已经在脑海里"冲泡"了一次。

---

## 二、数据来源

### 2.1 CQI 咖啡品质数据库（主数据源）

**Coffee Quality Institute（CQI）** 的品测数据，经 Q-Grader 专业评分，是目前最权威的免费咖啡数据集。

| 信息 | 数据 |
|------|------|
| 来源 | [jldbc/coffee-quality-database](https://github.com/jldbc/coffee-quality-database) |
| 阿拉比卡 | ~1300 条 |
| 罗布斯塔 | ~28 条 |
| 字段数 | 44 个 |
| 许可证 | 开源免费 |

**核心字段：**

| 分类 | 字段 | 示例值 |
|------|------|--------|
| 产地 | `Country.of.Origin` | Ethiopia |
| 产区 | `Region` | Guji-Hambela |
| 庄园 | `Farm.Name` / `Producer` | METAD PLC |
| 品种 | `Variety` | Geisha, Bourbon, Typica, Caturra… |
| 处理法 | `Processing.Method` | Washed / Wet, Natural, Honey… |
| 海拔 | `altitude_mean_meters` | 1950-2200m |
| 风味评分 | `Aroma` | 8.67 / 10 |
| | `Flavor` | 8.83 / 10 |
| | `Aftertaste` | 8.67 / 10 |
| | `Acidity` | 8.75 / 10 |
| | `Body` | 8.50 / 10 |
| | `Balance` | 8.42 / 10 |
| | `Sweetness` | 10 / 10 |
| | `Uniformity` | 10 / 10 |
| | `Clean.Cup` | 10 / 10 |
| 总分 | `Total.Cup.Points` | 90.58（精品门槛 80+）|

**下载链接：**

```
https://raw.githubusercontent.com/jldbc/coffee-quality-database/master/data/arabica_data_cleaned.csv
https://raw.githubusercontent.com/jldbc/coffee-quality-database/master/data/robusta_data_cleaned.csv
```

**用法**：一次性导入数据库，与 TMDB 拉取电影数据流程一致。

### 2.2 Open Food Facts API（产品信息补充）

- 免费开放 REST API
- 包含咖啡产品的品牌、包装、认证标签（有机/公平贸易/雨林联盟）
- 端点：`https://world.openfoodfacts.org/cgi/search.pl?search_terms=coffee&json=1`

### 2.3 SampleAPIs Coffee（咖啡饮品百科）

- `https://api.sampleapis.com/coffee/hot`
- 返回经典咖啡饮品定义（拿铁、卡布奇诺、美式等）
- 适合做 "饮品百科" 页面

### 2.4 用户共创（持续增长）

- 用户风味笔记
- 用户上传购买链接、价格
- 用户冲煮参数分享（粉水比、水温、研磨度）
- 盲测记录

### 2.5 可选爬虫补充（非必须）

- 各烘焙品牌官网产品页
- Coffee Review 专业测评

> **类比 TMDB 体系：**
> - CQI 数据库 = TMDB 的电影元数据（基础底座）
> - Open Food Facts = TMDB 的外部链接/公司信息
> - 用户共创 = TMDB 的用户评分/评论/列表

## 三、功能模块设计

### 3.1 🏗️ 咖啡豆档案页（核心）

每款豆子一个专属页面，信息密度对标 IMDb：

| 信息层级 | 内容 |
|----------|------|
| **基本信息** | 产地国家 · 产区 · 庄园 · 生产者 |
| **品种信息** | 品种名（Geisha / Bourbon / Typica…） |
| **处理工艺** | 处理法（日晒/水洗/蜜处理/厌氧…）+ 步骤说明 |
| **种植数据** | 海拔（m）· 采收年份 |
| **风味评分** | SCA 标准九维度评分 + 总分（雷达图可视化）|
| **风味轮** | 交互式风味轮导航，点击筛选同类豆子 |
| **烘焙度** | 浅 / 中 / 深 + 推荐烘焙度 |
| **认证标签** | 有机 · 公平贸易 · 雨林联盟 · 直接贸易 |

### 3.2 🎧 虚拟品鉴体验（"咖啡的预告片"）

由于无法直接试喝，用以下方式**引导用户建立感官期待**：

- **风味引导卡片** — 结构化描述，附常见类比
  - 香气：果香（柑橘/莓果）→ 花香（茉莉/玫瑰）→ 坚果/巧克力
  - 酸质：明亮的柠檬酸 vs 柔和的苹果酸
  - 余韵：红茶单宁感 vs 黑巧克力的苦甜
- **气味联想库** — 每种风味关联 1-2 个日常类比物
- **对比工具** — 分屏并排对比两款豆子的评分维度
- **🎵 风味音景（DemoNet 特色）** — 每款豆子配一段 ambient 音频，帮助用户进入风味情绪（雨林鸟鸣 + 研磨声 + 对应产地音乐片段）

### 3.3 📖 冲泡指南

- **研磨度声音播放器** — 点击收听不同研磨度的声音（粗盐 vs 细砂糖 vs 面粉状）
- **冲煮参数计算器** — 输入粉量 → 自动算出水量/水温/时间/粉水比
- **30-60s 冲煮短视频**（类比影视预告片）
- **设备兼容性推荐** — "这款豆子配 V60 口感最干净，配爱乐压更醇厚"

### 3.4 🗺️ 咖啡地图（实体层）

- **烘焙品牌档案** — 风格标签 + 推荐豆单
- **线下门店** — 地图 POI，用户标记去过/想去
- **时令日历** — 🌱 什么季节喝什么产区
  - 哥斯达黎加 1-3 月采收 → 5-7 月最佳品鉴期
  - 埃塞俄比亚 10-12 月采收 → 次年 2-4 月最佳

### 3.5 🎮 用户共创 & 游戏化（"咖啡的 Demo"）

| 功能 | 说明 |
|------|------|
| **风味检测** 🧪 | 用户喝完后提交风味笔记，与 CQI 评分对比 |
| **盲测挑战** | 两款匿名豆子，猜产地/品种/处理法 |
| **咖啡日记** | 记录今天的冲煮方案 + 感受（类似 Goodreads）|
| **成就系统** | "尝遍三大产区"、"解锁 10 种处理法"、"30 天手冲挑战"|

### 3.6 💳 收藏 & 购买

- **心愿单** — "想喝" 清单
- **购买链接聚合** — 一键跳转淘宝/京东/品牌官网（类比 Steam/Epic 购买入口）
- **价格追踪** — 历史价格曲线 + 降价通知

## 四、技术实现建议

### 4.1 数据导入流程

```
CQI CSV 文件
    ↓ 解析 + 清洗
数据库（PostgreSQL / SQLite / 你的选择）
    ↓ API 层
Demonet Coffee API
    ↓
前端咖啡档案页
```

### 4.2 推荐技术栈

- **后端**：延续 Demonet 现有框架
- **数据层**：CQI 一次导入 + 用户数据实时写入
- **前端雷达图**：Chart.js / D3.js / ECharts
- **风味轮**：可参考 [Coffee Flavor Wheel](https://notbadcoffee.com/flavor-wheel/) 做 Vue/React 组件

### 4.3 数据量估算

| 阶段 | 数据规模 | 说明 |
|------|---------|------|
| 起步 | ~1300 条 | CQI 阿拉比卡 + 罗布斯塔 |
| 增长 | +N 条/月 | 用户提交 + 手工录入新批次 |
| 成熟 | 万级 | 社区共创成为主要增长动力 |

---

## 五、一句话总结

> 影视有预告片 → 咖啡有**风味预告片**（风味轮雷达图 + 音景 + 对比工具）  
> 游戏有 Demo → 咖啡有**虚拟品鉴 + 用户共创的味觉笔记**  
> 音乐可试听 → 咖啡有**多维度的结构化风味对比体验**

用 **CQI 数据库做地基 + Open Food Facts API 做补充 + 用户共创做增长**，让咖啡板块成为 Demonet 上信息密度最高、体验最沉浸的栏目之一。
