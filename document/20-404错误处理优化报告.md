```java
@ExceptionHandler(NoResourceFoundException.class)
public ResponseEntity<ApiResponse<Void>> handleResourceNotFound(...) {
    // ✅ 返回 404 而非 500
    // ✅ 结构化日志（event + path + resource_type + method）

```

**设计亮点**：
- **优先级保证**：在 `RuntimeException` 处理器之前定义，确保优先匹配
- **结构化日志**：`event=static_resource_not_found` + 固定字段，符合 Observability Skill
- **计数器指标**：按资源类型分组（`upload`, `actuator`, `swagger`, `other`），避免高基数问题
- **友好的错误信息**：根据路径前缀定制提示（"文件不存在"/"监控端点不存在"/"资源不存在"）

#### 2. **资源类型分类**

```java
private String classifyResourceType(String path) {
    if (path.startsWith("/uploads/") || path.startsWith("/uploads-")) return "upload";
    else if (path.startsWith("/actuator/")) return "actuator";
    else if (path.startsWith("/swagger-ui/") || path.startsWith("/v3/api-docs/")) return "swagger";
    else return "other";
}
```

**符合 Observability Skill 原则**：
- ✅ 使用固定、小基数的标签值
- ✅ 避免使用用户 ID、完整 URL、错误信息作为标签
- ✅ 计数器 key 格式：`metrics:http_404_total:{resource_type}`

#### 3. **智能后备图片**

```javascript
const fallbackImages = {
  default: 'https://placehold.co/600x400/1a1a2e/808080?text=No+Image',

  poster: 'https://placehold.co/300x450/1a1a2e/808080?text=No+Poster',
```

#### 4. **自动检测后备类型**

```javascript
function detectFallbackType(src) {
  if (src.startsWith('/uploads/')) return 'upload'
  if (src.includes('poster')) return 'poster'
  if (src.includes('cover')) return 'cover'
  return 'default'
}
```

### **修改文件**
- `frontend/src/api/request.js` — 拦截器静默处理 404

#### **静默降级策略**

```javascript
// 404 静态资源错误 — 静默降级，不弹窗打扰用户
if (error.response?.status === 404 && isStaticResourceError(error.config?.url)) {
  if (import.meta.env.DEV) {
    console.debug('[RequestInterceptor] Static resource not found:', error.config.url)
  }
  return Promise.reject(error) // 静默返回，不弹窗
}
```

**符合 Frontend UI Engineering Skill**：
- ✅ 避免不必要的错误弹窗打扰用户
- ✅ 开发环境记录 debug 日志
- ✅ 区分业务错误和静态资源错误

### **集成示例**（已应用到 GameCard.vue）
