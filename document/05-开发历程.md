- **后台进程管理故障**：
  - `bash async:true` 启动的后端因 300 秒硬超时被系统 `taskkill`，导致用户登录失败。
  - 教训：`mvn spring-boot:run` 必须用 `&` 后台启动或无超时限制，生产环境应使用 `java -jar`。

## 注册安全与提示系统重构（2026-07-03 续）

### 注册防护三层机制
- **IP 限流**：`AuthController.register()` 使用 Redis `INCR + EXPIRE`，同 IP 1 分钟最多 3 次
- **Cloudflare Turnstile**：前端 Register.vue 动态加载 widget → 后端 `POST siteverify` 验证

## Spring Boot 4.0 Jackson 3 迁移（2026-07-05）

### 问题现象

升级 Spring Boot 至 4.0.6 后，后端启动失败：

```
Parameter 0 of constructor in com.example.demonet.config.ResponseAdvice
required a bean of type 'com.fasterxml.jackson.databind.ObjectMapper'
that could not be found.
```

### 根因

Spring Boot 4.0 将默认 JSON 库从 **Jackson 2** 升级到 **Jackson 3**（版本 3.1.2）：

- 核心包名变更：`com.fasterxml.jackson.core/databind` → `tools.jackson.core/databind`（注解包 `com.fasterxml.jackson.annotation` 保持不变）
- Spring Boot 4.0 的自动配置**只注册 Jackson 3 的 `ObjectMapper` bean**（新包名），不再注册 Jackson 2 的
- 项目代码引用的是 Jackson 2 的 `ObjectMapper`，因 `pom.xml` 显式声明 `jackson-datatype-jsr310` 传递依赖拉入了 Jackson 2 的 `jackson-databind`，所以**能编译通过但运行时找不到 bean**

### Jackson 2 → Jackson 3 关键变化

| 维度 | Jackson 2 | Jackson 3 |
|------|-----------|-----------|
| 包名 | `com.fasterxml.jackson.*` | `tools.jackson.*`（注解包不变） |
| ObjectMapper | 可变，`new ObjectMapper()` | **不可变**，必须 `JsonMapper.builder().build()` |
| 异常 | `JsonProcessingException`（受检） | `JacksonException`（非受检，继承 RuntimeException） |
| Java 时间 | 需手动注册 `JavaTimeModule` | **内嵌支持**，无需注册 |
| DefaultTyping | `ObjectMapper.DefaultTyping`（内部类） | `DefaultTyping`（独立枚举） |
| 未知字段 | 默认 `FAIL_ON_UNKNOWN_PROPERTIES=true` | 默认 `false`（更宽容） |
| 日期序列化 | 默认时间戳 | 默认 ISO 格式（`WRITE_DATES_AS_TIMESTAMPS=false`） |

### 修复内容（7 个文件）

1. **`pom.xml`**：移除 `jackson-datatype-jsr310` 依赖（Jackson 3 已内嵌 Java 时间支持）
2. **`ResponseAdvice.java`**：import 包名迁移 + `JsonProcessingException` → `JacksonException`
3. **`ItemService.java`**：`ObjectMapper` import 包名迁移
4. **`IGDBService.java`**：import 迁移 + `new ObjectMapper()` → `JsonMapper.builder().build()`
5. **`AdminController.java`**：`ObjectMapper` 和 `TypeReference` 全限定类名迁移
6. **`FetchConsumer.java`**：改为注入 Spring 容器的 ObjectMapper bean（替代每次 `new`）
7. **`RedisConfig.java`**：
   - `JsonMapper.builder()` 构建 + `activateDefaultTyping()` 链式调用
   - `DefaultTyping`（独立枚举）
   - `GenericJackson2JsonRedisSerializer` → `GenericJacksonJsonRedisSerializer`

### 踩坑记录

- **`GenericJackson3JsonRedisSerializer` 类名不存在**：最初按命名规律推测类名为 `GenericJackson3JsonRedisSerializer`，但通过 `javap` 检查 Spring Data Redis 4.0.5 的 jar 发现实际类名是 **`GenericJacksonJsonRedisSerializer`**（不带 "3"），构造函数签名为 `GenericJacksonJsonRedisSerializer(tools.jackson.databind.ObjectMapper)`。
- **Redis 缓存不兼容**：切换序列化器后，Redis 中旧缓存数据（Jackson 2 格式）无法反序列化，需执行 `redis-cli FLUSHDB` 清空。
- **`@JsonIgnore` 无需修改**：`User.java` 中的 `@JsonIgnore` 注解位于 `com.fasterxml.jackson.annotation` 包，Jackson 3 保持不变。
