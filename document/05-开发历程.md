# 开发历程

> 阶段 0 → 阶段 10，完整开发记录

---

## 阶段 0：项目骨架

- 初始化 Spring Boot 4 + Vue 3 项目
- Docker Compose 编排 MySQL + Redis + RabbitMQ
- 前端 Vite + Tailwind CSS 4

## 阶段 1：CRUD API

- ItemController 实现 RESTful CRUD
- MyBatis-Plus 分页查询
- 自定义 slug 生成逻辑

## 阶段 2：用户系统

- Spring Security + JWT 无状态认证
- 注册/登录/获取当前用户
- 密码 BCrypt 加密，默认管理员 `admin / changeme`

## 阶段 3：搜索与推荐

- 标签系统（tags + item_tag_mapping 多对多）
- 按品类/标签/关键词搜索
- 热门推荐列表排序

## 阶段 4：异步抓取管道

- RabbitMQ 消息队列 + FetchConsumer
- SteamService / TMDBService 外部 API 集成
- Python 脚本批量导入种子数据

## 阶段 5：品类扩展

- 从 4 品类扩展到 8 品类 → 最终 10 品类
- 每种品类独立类型标识和 info_json 元数据结构

## 阶段 7.0：前置准备

- ALTER TABLE 加 `media_url`、`external_link` 列
- 创建 `constants/types.js`（10 品类 emoji/accent/heroColor/light）
- 创建 `styles/transitions.css`（品类专属动画）
- 种子数据补封面 + 外部链接

## 阶段 7.1：游戏频道

- **GameHero**：无限循环轮播（3 份全量拷贝 + GPU 加速 `translateX`）
- **GameCard**：竖版卡片 + DEMO 彩虹脉冲徽章 + 内容滑入视差
- Steam 数据管道：Python 脚本调用 Store API 拉取 14 款游戏真实封面
- 双视频源（B站 + YouTube）iframe 嵌入，可切换
- 试玩链接：5 款有 `steam://install/APPID`

## 阶段 7.2：电影频道

- **MovieHero**：全屏 Banner（`h-[calc(100vh-4rem)]`），5s 间隔交叉淡入
- **MovieCard**：1/3 海报 + 2/3 信息横卡，hover 海报放大 110%
- TMDB 数据管道：16 部电影真实海报+背景，15 部有 YouTube 预告片
- Video 获取：`language=zh-CN` 会屏蔽 YouTube 预告片，需不带 language 参数单独请求

## 阶段 9：质量加固（已完成 9 项）

1. **S1 — SQL 注入修复**：`listRecommended()` 改用 `JdbcTemplate` 参数化查询
2. **A1 — 全局异常处理**：`GlobalExceptionHandler`（DuplicateKey→409, AccessDenied→403, RuntimeException→400, 兜底→500）
3. **A3 — passwordHash @JsonIgnore**：登录/注册响应不再包含哈希
4. **B1 — 401/403 重定向修复**：axios 拦截器 + router beforeEach 处理
5. **B2 — Tailwind 动态类名修复**：types.js 存储完整类名
6. **B4 — SearchBar 回车防抖修复**：回车立即搜索
7. **S2/S3 — Admin 访问控制**：前端路由守卫 + 后端 SecurityConfig ADMIN 角色校验
8. **S4 — 敏感配置环境变量化**：全部改为 `${ENV_VAR:default}` + .env.example
9. **编辑器全品类专有化**：11 个专属编辑器 + 视频链接智能识别
