# 旧版 0.1 与新版 1.0 对比分析与迁移建议

> **分析日期**：2026-05-28
> **参考对象**：`参考项目DemoNet-0.1/`（静态 HTML + Bootstrap + jQuery）
> **对比对象**：DemoNet 1.0（Vue 3 + Spring Boot 4 + Tailwind CSS）

---

## 一、视觉与交互对比

### 1.1 首页（index.html vs Home.vue）

| 特性 | 旧版 0.1 | 新版 1.0 | 评价 |
|------|---------|---------|------|
| 顶部轮播 | Bootstrap Carousel，**100vh 全屏**，带标题+描述+链接 | 无全屏轮播，使用单个 `h-72` banner 卡片 | **旧版更优**：全屏沉浸式轮播是"策展化"体验的核心入口 |
| 导航栏 | 透明底 → 滚动后变暗色（`setupScrollEffect`） | 固定暗色底，无滚动变化 | **旧版更优**：透明→实色的过渡更有层次感 |
| 热门推荐 | **3D 翻转卡片**（`component.css`），海报→翻转→信息面板 | 普通 grid 卡片（`AppCard`），emoji + 文字 | **旧版显著更优**：3D 翻转是"策展化"最直观的展示 |
| 品类入口 | 无（旧版只有游戏/电影/动漫三个频道页） | 有品类网格（8 个品类 tile） | **新版更优**：品类导航更完整 |
| 整体氛围 | 白底 + 彩色内容，明暗交替 | 全暗色底（`bg-gray-950`） | 各有特点，但旧版白底让内容更突出 |

### 1.2 游戏频道（game.html vs GameHero + GameCard）

| 特性 | 旧版 0.1 | 新版 1.0 | 评价 |
|------|---------|---------|------|
| 顶部轮播 | Swiper 双区联动：**上方 400px 大图 + 左侧 250px 标题导航栏**，点击左侧标题切换右侧大图 | 横向滚动轮播（280/320px 卡片），底部圆点分页 | **旧版更优**：左侧标题导航+右侧大图的联动模式信息密度更高，视觉冲击力更强 |
| 卡片样式 | **CSS 3D 渐变揭示**：背景图覆盖，hover 时背景上移、暗色渐变下移、内容从底部滑入，带精细的 cubic-bezier 缓动 | hover 滑动揭示（`transitions.css`），内容阶梯式淡入 | **接近**：新版已复刻了旧版的核心动画逻辑，但旧版的 `:before/:after` 伪元素双图层更精致 |
| 卡片网格 | CSS Grid 2/4 列自适应 | 普通 grid，与游戏频道卡片样式一致 | **持平** |
| 卡片字体 | 引入 Google Fonts（Rubik + Cardo），衬线斜体描述文字 | 系统默认字体 | **旧版更优**：专用字体提升了品质感 |
| DEMO 标记 | 无 | 有（`animate-pulse` 脉动徽章） | **新版更优** |

### 1.3 电影频道（movie.html vs MovieHero + MovieCard）

| 特性 | 旧版 0.1 | 新版 1.0 | 评价 |
|------|---------|---------|------|
| 轮播效果 | **三层动画系统**：①背景模糊层（`blur(1.5vh)`）②前景缩放层（`scale(0.75→0.8)`）③内容淡入层，配合 `translateX` 滑入/滑出，每次切换有 1350ms 的分层延迟序列 | 简单背景交叉淡入（900ms `transition-all`） | **旧版显著更优**：多层分时动画是电影频道的"影院级"体验核心 |
| 内容展示 | 超大标题（`9vh`）+ 英文副标题（`6vh`）+ "next"箭头链接 + 详情链接 | 标题（`5xl/7xl`）+ 描述 + 两个按钮 | **旧版更优**：中英双语标题更有电影质感 |
| 切换动画 | `@keyframes hack`：当前幻灯片向左滑出 → 透明度切换 → 从右侧重入 | 无切换动画，仅背景交叉淡入 | **旧版显著更优** |
| 链接悬停 | "next" 箭头 hover 时：横线拉长 1.5 倍 + 箭头向右滑动 6vh | 无此效果 | **旧版更优**：精致的微交互 |
| 卡片样式 | **3D 翻转书卡**（`component.css`）：260×400px 竖版海报，hover 旋转 -78° 露出背面信息面板（导演/评分/简介），带 box-shadow 模拟光影 | 横向宽卡（1/3 图 + 2/3 文字），hover 图片微放大 | **旧版显著更优**：3D 翻转海报是电影频道最具辨识度的"策展化"效果 |

### 1.4 动漫频道（anime.html，仅旧版有）

| 特性 | 旧版 0.1 | 新版 1.0 | 评价 |
|------|---------|---------|------|
| 卡片样式 | `profile-card-2`：图片卡片 + hover 灰度化 + 标题上移 + 标签上移 | 尚未实现 | 旧版已有基础实现 |
| 图片交互 | hover 时 `filter: grayscale(100%)`，标题和标签同步上移 10px | — | 效果简单但有记忆点 |

### 1.5 详情页（detail.html vs Detail.vue）

| 特性 | 旧版 0.1 | 新版 1.0 | 评价 |
|------|---------|---------|------|
| 布局 | Bootstrap 栅格：左 4 列封面图 + 右 8 列文字 | 顶部视频播放器 + 下方元数据面板 | **新版更优**：视频优先的布局符合"体验前置"定位 |
| 配置要求 | 有独立的最低/推荐配置面板（`minSpec` / `recSpec`） | 无独立配置面板 | **旧版更优**：硬件配置是玩家决策的关键信息 |
| "关于这款游戏" | 有独立的长文本段落（`about`） | 无独立字段 | **旧版更优** |

## 二、数据结构对比

### 2.1 游戏数据（games.json vs items 表）

| 旧版字段 | 新版对应 | 差异分析 |
|----------|---------|---------|
| `id`（字符串 slug） | `slug` | 新版分离了 `id`（自增）和 `slug`（路由标识），更规范 |
| `title` | `title` | 一致 |
| `cover` | `cover_url` | 一致 |
| `cardCover` | 无直接对应 | **丢失**：旧版为卡片、轮播、封面分别指定图片 |
| `swiperCover` | `wide_cover_url` | 近似对应 |
| `cardTitle` / `cardDesc` | 无 | **丢失**：旧版允许卡片显示与详情页不同的短标题/短描述 |
| `swiperName` | 无 | **丢失**：轮播导航中显示的简称 |
| `swiperOrder` | 无 | **丢失**：控制轮播展示顺序 |
| `shortDesc` | 无 | **丢失**：列表页用的简短描述 |
| `description` | `description` | 一致 |
| `info`（对象） | `info_json`（JSON） | 新版更灵活 |
| `about` | 无 | **丢失**：游戏详细介绍长文本 |
| `minSpec` / `recSpec` | 无 | **丢失**：硬件配置要求 |
| `category` | `type` | 一致 |

**旧版 `info` 对象字段**（游戏）：
```json
{
  "发行日期": "2020.11.5",
  "开发商": "Respawn Entertainment",
  "发行商": "Electronic Arts",
  "最新测评": "特别好评"
}
```
新版 `info_json` 对应字段：`developer`, `genre`, `platform`, `demo_url`, `videos` 等，**字段更丰富但缺少发行商和测评状态**。

### 2.2 电影数据（movies.json vs items 表）

| 旧版字段 | 新版对应 | 差异分析 |
|----------|---------|---------|
| `cover`（竖版海报） | `cover_url` | 一致 |
| `wideCover`（宽幅横图） | `wide_cover_url` | 一致 |
| `stillCover`（剧照） | 无 | **丢失**：独立剧照字段 |
| `englishTitle` | 无 | **丢失**：英文原名对电影频道非常重要 |
| `slideOrder` | 无 | **丢失**：控制首页/电影轮播顺序 |
| `info`（对象） | `info_json`（JSON） | 新版字段不同 |

**旧版电影 `info` 字段**：导演、编剧、主演、类型、制片国家/地区、语言、上映日期、片长 — 共 8 个字段。
**新版电影 `info_json` 字段**：`director`, `year`, `duration`, `videos` — 仅 4 个字段，**缺少编剧、主演、类型、制片国家、语言**。

### 2.3 首页专属数据

旧版有两个新版不存在的数据文件：

- **`carousel.json`**：首页全屏轮播数据（5 条，跨品类混合），含 `type`、`id`、`image`、`title`、`desc`
- **`featured.json`**：首页 3D 翻转卡片数据（9 条，跨品类混合），含 `posterImg`、`headerImg`、`title`、`year`、`rating`、`duration`、`description`

这两个数据文件体现了一个重要设计理念：**首页内容是人工策展的，跨品类混合推荐**，而非简单的"热门列表"。

### 2.4 动漫数据（anime.json）

| 旧版字段 | 新版对应 | 差异分析 |
|----------|---------|---------|
| `displayTitle` | 无 | **丢失**：卡片显示的简称（如"全职高手" vs "全职高手之巅峰荣耀"） |
| `tag`（国漫/日漫/美漫） | 无 | **丢失**：产地分类标签 |
| `cardImg` | 无 | **丢失**：卡片专用图片 |

## 四、核心发现：重构中丢失的"策展化"灵魂

旧版虽然技术简陋（jQuery + 静态 HTML），但在"策展化"展示上有三个核心设计理念在重构过程中丢失了：

### 4.1 首页是"编辑精选"而非"算法推荐"

旧版首页通过 `carousel.json` 和 `featured.json` 两个文件**手工策展**了跨品类的推荐内容。新版首页仅使用 `getHotItems()` 和 `getRecommended()` 两个 API 按时间/收藏数排序，本质上是算法驱动而非人工策展。

**建议**：新增 `is_featured` 字段或 `featured_order` 排序字段，让管理员可以手工置顶首页轮播和精选推荐位。

### 4.2 每个频道是"不同的展厅"

旧版的三个频道有截然不同的视觉语言：
- 游戏 → Swiper 双区联动 + 渐变揭示卡片
- 电影 → 三层分时动画 + 3D 翻转海报
- 动漫 → 明亮卡片 + 灰度 hover

新版的游戏和电影频道虽然已有差异化的 Hero/Card 组件，但差异程度不如旧版鲜明。特别是电影频道丢失了最多的独特效果。

### 4.3 内容有多层"展示粒度"

旧版为同一内容提供了多层展示文案：
- `swiperName`（轮播导航简称，3-15 字符）
- `cardTitle` / `cardDesc`（卡片标题/描述）
- `shortDesc`（列表页短描述）
- `description`（详情页完整描述）
- `about`（游戏介绍长文本）

新版只有一个 `title` 和 `description`，所有场景使用同一份文案。

## 五、迁移路线图建议

```
阶段 A（当前 → 电影频道升级）
├── A1. MovieHero 多层切换动画改造
├── A2. MoviePosterCard 3D 翻转海报卡片
├── A3. 电影 info_json 补全（original_title, screenwriter, cast, genre, country）
├── A4. Home 新增全屏轮播组件
└── A5. Home 新增 3D 翻转精选推荐区

阶段 B（数据模型增强）
├── B1. items 表增加 poster_url、featured_order 字段
├── B2. info_json 增加 card_title/card_desc/short_desc/about/min_spec/rec_spec
├── B3. DataImportService 更新，支持新字段导入
└── B4. 后端新增 GET /api/items/featured 接口

阶段 C（交互细节打磨）
├── C1. AppNavbar 滚动变色
├── C2. MovieHero 中英双标题
├── C3. Detail 游戏配置面板 + "关于"段落
├── C4. Google Fonts 引入
└── C5. 通用箭头 hover 动画工具类
```
