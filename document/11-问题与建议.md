## 阶段 8 修复 (2026-05-29)

### 69. @Cacheable 导致轮播/列表 API 失败（Redis 序列化）

- **时间**：2026-05-29
- **现象**：阶段 8 加 `@Cacheable` 后，hot/featured/recommended API 报错，Hero 组件 `catch {}` 静默吞错，所有频道轮播空白。
- **根因**：
  1. `GenericJackson2JsonRedisSerializer` 不支持 `LocalDateTime` 序列化，需要 `jackson-datatype-jsr310`
  2. Redis 不可用时缓存操作抛异常，无降级处理
- **解决**：
  - `pom.xml` 加 `jackson-datatype-jsr310`
  - `RedisConfig`：`ObjectMapper` 注册 `JavaTimeModule` → 传给 `GenericJackson2JsonRedisSerializer`
  - `RedisConfig`：实现 `CachingConfigurer.errorHandler()`，Redis 异常时只 warn 不抛，方法降级直查数据库

### 70. SpringDoc Swagger UI Safari 兼容

- **时间**：2026-05-29
- **现象**：Safari 访问 `/api/docs-ui` 下载为空文档或 HTML 文件，不渲染 Swagger UI 页面。
- **根因**：自定义路径 `/api/docs-ui` 在 Safari 中有重定向兼容问题 + SecurityConfig 未对 `/swagger-ui/**` 放行。
- **解决**：去掉自定义 springdoc 路径，使用默认路径 `http://localhost:8080/swagger-ui/index.html`，SecurityConfig 已放行 `/swagger-ui/**` 和 `/v3/api-docs/**`。

### 71. ReviewSection.vue 导入路径错误

- **时间**：2026-05-29
- **现象**：Vite 报 `Failed to resolve import "../../stores/auth" from "src/components/ReviewSection.vue"`
- **根因**：组件在 `src/components/`，写成了 `../../stores/auth`（往上了两级到 src 外层），正确路径是 `../stores/auth`
- **解决**：修正为 `../stores/auth` 和 `../api/review`

## 阶段 9 修复 (2026-06-09)

### 72. S1 — SQL 注入修复
- `ItemService.listRecommended()` 和 `getItemIdsByTags()` 改用 `JdbcTemplate.query(sql, PreparedStatementSetter, RowMapper)` 参数化查询

### 73. A1 — 全局异常处理
- 新建 `GlobalExceptionHandler`：`DuplicateKeyException` → 409、`AccessDeniedException` → 403、`RuntimeException` → 400、兜底 → 500

### 74. A3 — passwordHash @JsonIgnore
- `User.java` 的 `passwordHash` 加 `@JsonIgnore`，登录/注册响应不再包含哈希

### 75. B1 — 401/403 重定向修复
- axios 拦截器增加 403 处理，假 token 返回 403 时同样清 token + 跳转 `/login`
- router `beforeEach` 在 localStorage token 丢失时调用 `auth.logout()` 清除 Pinia 状态

### 76. B2 — Tailwind 动态类名修复
- `types.js` 新增 `pagBg` 字段返回完整类名（如 `bg-emerald-600`）
- `List.vue` 分页按钮改用 `meta.pagBg` 替代模板字符串拼接

### 77. B4 — SearchBar 回车绕过防抖 + 图标居中
- 回车键 `@keydown.enter` 改为 `clearTimeout(timer); searchNow()` 立即搜索
- 搜索图标和清除按钮改用 `top-1/2 -translate-y-1/2` 垂直居中

### 78. S2/S3 — Admin 访问控制
- 前端：`/admin` 路由已有 `requiresAuth + requiresAdmin` 守卫
- 后端：`SecurityConfig` 改为只放行 `/api/auth/login` 和 `/api/auth/register`，`/api/auth/me` 需认证 — 修复未登录时 `auth.getPrincipal()` NPE
- 注册：`email` 未传时自动生成 `{username}@demo.local`，修复 `Field 'email' doesn't have a default value`

### 79. S4 — 敏感配置环境变量化
- `application.yml` 全部敏感值改为 `${ENV_VAR:default}` 格式
- 新建 `.env.example` 模板文件

### ⚠️ 83. 详情页编辑器弹窗：Select 下拉框打开前为白色
- **现象**：品类、标签等 select 在未点击时白底，点开后才变暗
- **根因**：Element Plus `el-select` CSS 变量 `--el-fill-color-blank`（默认白色）覆盖 `background-color`
- **已尝试**：`--el-fill-color-blank: #374151` + `!important`
- **状态**：部分生效但仍有残留白色，不影响使用，待进一步排查
