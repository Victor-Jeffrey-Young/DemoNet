# 外部数据源集成

> **状态**：✅ 7 个在线 API + 1 个离线 CSV 数据源已集成
> **最后更新**：2026-07-09

---

## 概述

DemoNet 通过 7 个外部 API 和 1 个离线数据集获取多品类内容元数据，管理员在后台触发抓取后，数据经 RabbitMQ 异步消费、解析、入库为待审核 Item。

| 数据源 | 品类 | API 类型 | 认证方式 | 需配置 |
|--------|------|----------|----------|--------|
| Steam Store API | 游戏 | REST GET | 无需 Key | 否 |
| TMDB API v3 | 电影 / 动漫 | REST GET | API Key | `TMDB_API_KEY` |
| IGDB API v4 | 游戏 | REST POST | Twitch OAuth | `IGDB_CLIENT_ID` + `IGDB_CLIENT_SECRET` |
| AniList GraphQL | 动漫 | GraphQL POST | 无需认证 | 否 |
| Bangumi API | 动漫 | REST GET | 无需认证 | 否 |
| iTunes Search API | 音乐 | REST GET | 无需认证 | 否 |
| SteamGridDB API v2 | 游戏（海报补全） | REST GET | API Key | `STEAMGRIDDB_API_KEY` |
| CQI 咖啡品质数据库 | 咖啡 | 离线 CSV 导入 | 无 | 否 |

---

## 架构

```
Admin UI → POST /api/admin/fetch/{source}
  → RabbitMQ (demonet.fetch.{source})
    → FetchConsumer.handle{Source}Fetch()
      → {Source}Service.searchXxx() / fetchXxx()
        → 外部 API 请求（RestClient，5s/15s 超时）
      → ItemService.createItem() → MySQL (status=0 待审核)
      → FetchConsumer.autoTag() → 自动提取标签
```

所有外部 API 请求通过统一的 `RestClient` Bean（`RestClientConfig.java`），配置：
- `connectTimeout`: 5 秒
- `readTimeout`: 15 秒

### RabbitMQ 队列映射

| 队列名 | 数据源 | 路由键 |
|--------|--------|--------|
| `demonet.fetch.steam` | Steam | `fetch.steam` |
| `demonet.fetch.tmdb` | TMDB 电影 | `fetch.tmdb` |
| `demonet.fetch.tmdb-tv` | TMDB 电视剧 | `fetch.tmdb-tv` |
| `demonet.fetch.anilist` | AniList | `fetch.anilist` |
| `demonet.fetch.bangumi` | Bangumi | `fetch.bangumi` |
| `demonet.fetch.itunes` | iTunes | `fetch.itunes` |
| `demonet.fetch.igdb` | IGDB | `fetch.igdb` |

所有队列均配置 DLX 死信交换机（`demonet.dlx`）+ DLQ 死信队列（`demonet.dlq`），prefetch=5，重试 3 次。

## 1. Steam Store API

| 维度 | 详情 |
|------|------|
| **Base URL** | `https://store.steampowered.com/api` |
| **认证** | 无需 API Key（公开接口） |
| **请求方式** | GET |
| **Service** | `SteamService.java` |
| **队列** | `demonet.fetch.steam` |

### 端点

| 端点 | 用途 |
|------|------|
| `/appdetails?appids={id}&cc=cn&l=schinese` | 获取游戏详情（含描述/价格/截图/视频/需求/DLC） |
| `/storesearch/?term={q}&cc=cn&l=schinese` | 搜索 Steam 商店 |

### 主要方法

| 方法 | 说明 |
|------|------|
| `fetchByAppIds(List<Long>)` | 批量获取多个 App 详情 |
| `fetchAppDetail(Long)` | 获取单个 App 详情并构建 Item |
| `updateExistingGames()` | 回填手动录入游戏的 Steam 数据 |
| `backfillPosterUrls()` | 批量补全竖版海报 |
| `resolvePosterUrl(Long, String)` | 4 CDN 探测 → SteamGridDB → 回退横版（Redis 缓存 7 天） |

### 解析的 infoJson 字段

`developer` / `publisher` / `genre` / `platform` / `demo_available` / `free` / `is_dlc` / `release_date` / `languages` / `price` / `min_requirements` / `rec_requirements` / `dlc` / `features` / `screenshots` / `videos`

### 注意事项

- 语言参数 `l=schinese` 获取简体中文数据
- `supported_languages` 为 HTML 格式，需清理标签
- DLC 单独请求详情（最多 5 个），构建 JSON 数组
- 海报 URL 缓存到 Redis：`steam:poster:{appId}`，TTL 7 天
- ⚠️ 中国大陆环境 SSL 握手可能被中断，需代理

## 2. TMDB API v3

| 维度 | 详情 |
|------|------|
| **Base URL** | `https://api.themoviedb.org/3` |
| **图片 Base** | `https://image.tmdb.org/t/p` |
| **认证** | API Key（URL 参数 `api_key=`） |
| **配置** | `app.tmdb.api-key`（`.env`: `TMDB_API_KEY`） |
| **请求方式** | GET |
| **Service** | `TMDBService.java` |
| **队列** | `demonet.fetch.tmdb`（电影）/ `demonet.fetch.tmdb-tv`（电视剧） |

### 端点

| 端点 | 用途 |
|------|------|
| `/search/movie?api_key={k}&language=zh-CN&query={q}` | 搜索电影 |
| `/movie/{id}?api_key={k}&language=zh-CN&append_to_response=videos,credits` | 电影详情 + 预告片 + 演职员 |
| `/search/tv?api_key={k}&language=zh-CN&query={q}` | 搜索电视剧 |
| `/tv/{id}?api_key={k}&language=zh-CN&append_to_response=videos,credits` | 电视剧详情 |

### 主要方法

| 方法 | 说明 |
|------|------|
| `searchMovies(String)` | 搜索电影并获取详情，type=movie |
| `fetchMovieDetail(int)` | 获取电影详情并构建 Item |
| `searchTV(String)` | 搜索电视剧并获取详情，type=anime |
| `fetchTVDetail(int)` | 获取电视剧详情并构建 Item |

### 解析的字段

`title` / `overview` / `poster_path` → 封面 / `backdrop_path` → 横版 / `genres[].name` / `release_date` / `runtime` / `vote_average` / `credits.crew` → 导演 / `videos.results` → YouTube 预告片 / `created_by` → 电视剧创建者 / `networks` → 电视网 / `number_of_seasons` / `number_of_episodes`

### 获取凭证

1. 注册 [TMDB 账号](https://www.themoviedb.org/signup)
2. Settings → API → Request API Key
3. 选择 Developer 类型，填写应用信息

### 注意事项

- 预告片搜索需 `language=zh-CN`，但 YouTube 预告片可能被过滤，需不带 language 单独请求
- ⚠️ 中国大陆环境可能无法直接访问，需代理

## 3. IGDB API v4

| 维度 | 详情 |
|------|------|
| **Base URL** | `https://api.igdb.com/v4` |
| **Token URL** | `https://id.twitch.tv/oauth2/token` |
| **认证** | Twitch OAuth（Client Credentials flow） |
| **配置** | `IGDB_CLIENT_ID` + `IGDB_CLIENT_SECRET` |
| **请求方式** | POST（body 为类 GraphQL 查询语法） |
| **Service** | `IGDBService.java` |
| **队列** | `demonet.fetch.igdb` |

### 端点

| 端点 | 用途 |
|------|------|
| `/games` | 游戏搜索/详情（POST body 为查询语句） |

### 主要方法

| 方法 | 说明 |
|------|------|
| `searchGames(query, limit)` | 按名称搜索 |
| `fetchPopularGames(limit)` | 高评分热门（评分 ≥80 且人数 ≥50） |
| `fetchRecentGames(limit)` | 近一年新游戏（评分 ≥70） |
| `fetchGameById(igdbId)` | 按 ID 获取详情 |

### 查询字段

`name` / `summary` / `cover.url` / `first_release_date` / `genres.name` / `platforms.name` / `themes.name` / `game_modes.name` / `rating` / `rating_count` / `screenshots.url` / `videos.video_id`（YouTube）/ `websites.url`（Steam/Epic/GOG/官网）/ `similar_games.name` / `involved_companies.company.name`

### Token 机制

- 使用 Twitch Client Credentials flow 获取 access_token
- Token 缓存在实例字段，过期前 60 秒自动刷新
- `synchronized` 保证线程安全（单节点），多节点部署应迁移至 Redis（见[问题、待办与演进计划](08-问题与待办.md#历史方案附录)中的历史架构改造方案第 14 项）

### 获取凭证

1. 注册 [Twitch 账号](https://www.twitch.tv/)
2. 进入 [Twitch Developer Console](https://dev.twitch.tv/console)
3. Applications → Register Your Application
4. Name: `DemoNet`，OAuth Redirect URL: `http://localhost`
5. 获取 Client ID 和 Client Secret

### API 触发

```http
POST /api/admin/fetch/igdb
Authorization: Bearer <admin_jwt>

{
  "query": "Hades",
  "endpoint": "search",   // search / popular / recent
  "limit": 10,
  "targetType": "game"
}
```

### 查询结构

```graphql

  Page(perPage: 10) {
    media(search: $search, type: ANIME, sort: POPULARITY_DESC) {

      description
      coverImage { large extraLarge }
      bannerImage
      genres
      studios(isMain: true) { nodes { name } }
      seasonYear
      episodes
      averageScore
      synonyms
```

## 7. SteamGridDB API v2

| 维度 | 详情 |
|------|------|
| **Base URL** | `https://www.steamgriddb.com/api/v2` |
| **认证** | API Key（Header `Authorization: Bearer {key}`） |
| **配置** | `app.steamgriddb.api-key` 或管理后台 `app_settings` 表 |
| **请求方式** | GET |
| **Service** | `SteamGridDBService.java` |
| **队列** | 无（由 SteamService 内部调用，非独立抓取源） |

### 端点

| 端点 | 用途 |
|------|------|
| `/grids/steam/{appId}?dimensions=600x900` | 获取竖版海报列表 |

### 主要方法

| 方法 | 说明 |
|------|------|
| `findPosterUrl(Long appId)` | 获取竖版海报 URL |

### API Key 管理

- 优先从数据库 `app_settings` 表读取（管理后台可修改）
- 回退到环境变量 `app.steamgriddb.api-key`
- 5 分钟内存缓存，避免频繁查库

### 获取凭证

1. 注册 [SteamGridDB 账号](https://www.steamgriddb.com/)
2. Profile → Preferences → API Key

### 使用场景

`SteamService.resolvePosterUrl()` 在 4 个 Steam CDN 路径探测失败后，调用 SteamGridDB 获取社区贡献的竖版封面。结果缓存到 Redis（`steam:poster:{appId}`，TTL 7 天）。

## 8. CQI 咖啡品质数据库（离线 CSV）

| 维度 | 详情 |
|------|------|
| **数据源** | [jldbc/coffee-quality-database](https://github.com/jldbc/coffee-quality-database) |
| **数据量** | ~1300 条阿拉比卡品测数据 |
| **导入方式** | 离线 CSV 导入（`DataImportService.seedCQICoffeeIfMissing()`） |
| **导入条件** | 数据库无 `type=coffee AND source=cqi` 的记录时自动导入 |
| **文件位置** | `backend/src/main/resources/data/arabica_coffee.csv` |

### 导入逻辑

1. 启动时检查是否已有 CQI 数据
2. 读取 CSV，跳过 header 行
3. 解析 44 个字段，提取产地/品种/处理法/海拔/采收年/11 项 SCA 评分
4. 筛选 SCA 总分 ≥ 86 分的精品咖啡
5. 构建 Item（type=coffee, source=cqi）并入库

### infoJson 字段

`origin` / `region` / `farm` / `producer` / `variety` / `process` / `altitude` / `harvest` / `aroma` / `flavor` / `aftertaste` / `acidity` / `body` / `balance` / `uniformity` / `clean_cup` / `sweetness` / `cupper_points` / `total_cup_points`

### 注意事项

- CSV 首字段为空引号 `""`，所有列索引需 +1
- `flavor` 数值字段与风味描述冲突，描述改存为 `flavor_notes`
- 封面使用 `picsum.photos` 随机图（无真实咖啡图片源）
