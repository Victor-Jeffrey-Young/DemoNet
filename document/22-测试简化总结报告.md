## 📝 具体改进内容

### 1. 代码复用性

**新建 `TestAuthentication` 工具类**:

```java
// 之前:每个测试类重复 6 行
auth = mock(Authentication.class, withSettings().lenient());
when(auth.getPrincipal()).thenReturn(1L);

// 现在:一行调用
auth = TestAuthentication.withUserId(1L);
```

**收益**:
- 消除 2 个重复的 `mockAuth()` 方法 (12 行)
- 统一认证 mock 模式
- 支持扩展角色认证 (`withUserIdAndRole`, `withAdmin`)

---

**新建 `BaseIGDBServiceTest` 基类**:

```java
// 之前:每个测试方法重复 6 行 RestClient stub
when(restClient.post()).thenReturn(requestBodyUriSpec);
when(requestBodyUriSpec.uri(anyString())).thenReturn(requestBodyUriSpec);
// ...

// 现在:两行完成
mockTokenCache("cached-token");
mockRestClientChain(gameResponse);
```

**收益**:
- 消除每个测试方法中的重复 stub
- 提供语义化辅助方法 (`mockTokenCache`, `mockRestClientChain`, `mockEmptyResponse`)
- 统一 IGDB 测试 mock 模式

---
