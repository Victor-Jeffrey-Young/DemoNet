## 二、视觉设计

### 风格定位

| 属性 | 值 |
|------|-----|
| 主题 | 暖色木纹 × 桌垫纹理 |
| Hero 遮罩 | `from-amber-950 via-yellow-950/70 to-gray-950` |
| 主色调 | `amber-500` / `yellow-400`，辅以暖棕 |
| 氛围 | 桌面游戏店的温馨感，模拟木桌 + 毛毡垫的质感 |

### BoardgameHero — 暖色轮播

```
┌──────────────────────────────────────────────┐
│  [背景: wideCoverUrl 宽图 + 暖琥珀渐变遮罩]    │
│                                               │
│   🎲 桌游专区                                  │
│   Gloomhaven                                   │
│   史诗级地牢探索桌游。宏大世界观...               │
│   [查看详情]                                   │
│                                    ○ ○ ● ○ → │
└──────────────────────────────────────────────┘
```

- 数据源：`getFeatured({type:'boardgame'})` → fallback `getHotItems()`
- Vue Transition 交叉淡入淡出，5s 自动播放
- 从 `AnimeHero.vue` 复制模板，全局替换主题色

### BoardgameCard — 横版宽卡（核心）

```
┌──────────────────────────────────────────────────┐
│ ┌───────────┐  🎲 桌游                           │
│ │           │  Dune: Imperium                     │
│ │  盒图     │  👥 1-4人  ⏱ 60-120min  ⭐ 3.0/5  │
│ │ 1/3 宽    │  [工人放置] [牌库构筑] [策略]        │
│ │           │  沙丘帝国。工人放置+牌库构筑...       │
│ └───────────┘                     [▶ 规则教学] → │
└──────────────────────────────────────────────────┘
```

每次横版宽卡展示：

| 区域 | 内容 | 数据来源 |
|------|------|---------|
| 左侧 1/3 | 盒图/封面 | `wideCoverUrl \|\| coverUrl` |
| 右上 | 品类标签「桌游」 | 常量 |
| 第二行 | 人数 👥 + 时长 ⏱ + 重量 ⭐ | `info_json.players`/`playtime`/`weight` |
| 第三行 | 机制标签 | `item.tags`（从后端关联标签读取）或 `info_json.mechanics` |
| 第四行 | 描述摘要 | `description` (line-clamp-2) |
| 右下 | 规则教学 ▶ 按钮 | `info_json.videos.bilibili` 或 `info_json.videos.youtube` |

**重量可视化**：`weight` 值用 ★ 符号渲染（1.0→1★, 3.9→4★），直观展示复杂度。

## 三、展示策略（A + B 结合）

### A. 卡片规则视频入口 ▶

卡片右侧底部「▶ 规则教学」按钮，点击跳转 B站教学视频（`info_json.videos.bilibili`）或 YouTube（`info_json.videos.youtube`）。用户无需进入详情页即可观看教学。

### B. 核心机制标签 🏷️

从 `items` 的关联标签中读取桌游机制标签（如"工人放置""牌库构筑""合作""策略"），以彩色 chips 展示在卡片信息区。

种子数据已有的标签映射（`ITEM_TAGS`）：
```
dune-imperium → strategy, deck-building, worker-placement
gloomhaven → dungeon-crawl, coop, campaign, heavy
wingspan → engine-building, family, birds, medium
spiritisland → coop, strategy, complex, area-control
catan → family, trading, gateway, dice
...
```

这些标签将在卡片上作为可视化机制标签展示。

### C. 详情页桌游专区（后续优化）

详情页 `Detail.vue` 针对 `type=boardgame` 增加专用面板：
- 规则教学视频（`media_url`）
- 玩家人数 + 时长 + BGG 重量表
- 机制标签云
- 游戏简介

本次开发先实现 A + B，C 作为下一轮优化。
