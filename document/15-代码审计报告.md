## 二、安全问题（Security）

### S-1 JWT 密钥硬编码 — ✅ 已修复

- **文件**：`application.yml`
- **问题**：真实 JWT 密钥提交至 Git
- **修复**：改为环境变量注入 + 占位符默认值，Git 历史已清理

### S-2 登录接口无速率限制 — ✅ 已修复

- **文件**：`AuthController.java`
- **问题**：`login()` 无限流，可暴力破解
- **修复**：Redis 计数器，登录 5/min，失败 4 次后提示剩余次数

### S-3 docker-compose 端口暴露 — 🚧 待修复

- **文件**：`docker-compose.prod.yml`
- **问题**：MySQL/Redis/RabbitMQ 端口映射到宿主机，生产环境存在公网暴露风险
- **建议**：生产环境移除中间件端口映射，仅用 Docker 内部网络；如需管理访问用 SSH 隧道

### S-4 上传无扩展名白名单 — ✅ 已修复

- **文件**：`AdminController.java`
- **问题**：仅校验 Content-Type，扩展名来自用户输入，可上传 `.html`/`.svg` XSS 载荷
- **修复**：MIME 白名单 + 扩展名白名单（jpeg/png/webp/gif/pdf/epub）

### S-5 `fix_images.py` 硬编码管理员密码 — ✅ 已修复

- **文件**：`fix_images.py`
- **问题**：脚本中硬编码 `admin / changeme`
- **修复**：管理员密码改为 `ADMIN_DEFAULT_PASSWORD` 环境变量；脚本已加入 `.gitignore`

### S-6 Token 存储在 localStorage — 🚧 待修复

- **问题**：前端 JWT 存储在 `localStorage`，XSS 场景下可被窃取
- **建议**：迁移至 `httpOnly` cookie，或配合 CSP 策略降低 XSS 风险

### S-7 Cloudflare Turnstile 异常吞没 — 🚧 待修复

- **文件**：`AuthService.java`
- **问题**：Turnstile 验证 `catch (Exception e) {}` 空吞异常，网络超时时直接放行
- **建议**：至少记录日志，明确决定放行还是拒绝（见 [12-后端架构改造计划](12-后端架构改造计划.md) 第 17 项）

## 三、架构问题（Architecture）

### A-1 `AdminController` 过度臃肿 — 🚧 待修复

- **文件**：`AdminController.java`（26KB，20+ 端点，8+ 职责）
- **问题**：Item CRUD、文件上传、RabbitMQ Fetch（7 源）、轮播、标签、用户管理、统计、审批全部混在一个 Controller
- **建议**：拆分为 `AdminItemController`/`AdminTagController`/`AdminFetchController`/`AdminSettingController`/`AdminUserController`

### A-2 `DataImportService` 超大文件 — 🚧 待修复

- **文件**：`DataImportService.java`（57KB）
- **问题**：单个服务文件近 60KB，10 个数据源的导入逻辑全部混在一起
- **建议**：按数据源拆分独立 Service，或引入策略模式

### A-3 `KEYS` 命令阻塞 Redis — ✅ 已修复

- **文件**：`DemoNetApplication.java`
- **问题**：`redisTemplate.keys()` 阻塞 Redis 主线程
- **修复**：改用 `SCAN` 迭代删除（`scanKeys()` 方法）

### A-4 零测试覆盖 — 🚧 待修复

- **问题**：项目无任何单元测试或集成测试
- **建议**：优先为 `AuthService`/`ItemService`/`ReviewService`/`JwtTokenProvider` 编写测试（JUnit 5 + Mockito + Testcontainers）

### A-5 生产环境后端无持久化卷 — 🚧 待修复

- **文件**：`docker-compose.prod.yml`
- **问题**：backend 容器未挂载 `uploads` 目录，重启后用户上传的图片丢失
- **建议**：添加 `volumes: - uploads:/app/uploads`

### A-6 配置不一致：上传大小限制 — ✅ 已修复

- **问题**：`application.yml`（10MB）与 `application-example.yml`（100MB）不一致
- **状态**：已统一为 10MB（架构分析报告第 13 项建议降至此值）

### A-7 混合 ORM：MyBatis-Plus + JdbcTemplate — 🚧 待修复

- **文件**：`ItemService.listRecommended()`
- **问题**：同时使用 MyBatis-Plus 和 JdbcTemplate 手动 RowMapper
- **建议**：统一为 MyBatis-Plus `@Select` 注解或 XML mapper

### A-8 前端品类组件复制粘贴 — 🚧 待修复

- **文件**：10 个 `XxxCard.vue` + `XxxHero.vue`
- **问题**：GameCard/AnimeCard/MovieCard 等高度相似，差异仅在 icon/配色
- **建议**：提取通用 `CategoryCard.vue` + `CategoryHero.vue`，通过 props 差异化

## 四、代码质量（Quality）

### Q-1 `RuntimeException` 滥用 — ✅ 已修复

- **文件**：`AuthService.java`
- **问题**：所有业务错误抛 `RuntimeException`
- **修复**：引入 `BusinessException` + `ConflictException` 异常体系

### Q-2 `GlobalExceptionHandler` 400 兜底 — ✅ 已修复

- **文件**：`GlobalExceptionHandler.java`
- **问题**：所有 `RuntimeException` 返回 400，掩盖 500 错误
- **修复**：区分 `BusinessException`（400）和系统异常（500），系统异常记录完整堆栈

### Q-3 `ReviewController` 用户 ID 获取不一致 — 🚧 待修复

- **文件**：`ReviewController.java`
- **问题**：用 `auth.getName()` + `Long.valueOf()` 而非 `(Long) auth.getPrincipal()`
- **建议**：统一使用 `(Long) auth.getPrincipal()`

### Q-4 重复导入 `ObjectMapper` — ✅ 已修复

- **文件**：`ItemService.java`
- **修复**：Jackson 3 迁移时清理，改为注入全局 `ObjectMapper` bean

### Q-5 `@SneakyThrows` 滥用 — ✅ 已修复

- **文件**：`ResponseAdvice.java`
- **问题**：`@SneakyThrows` 绕过编译时异常检查
- **修复**：Jackson 3 迁移时改为 `JacksonException`（非受检异常），移除 `@SneakyThrows`

### Q-6 `UserItemController` 输入验证缺失 — ✅ 已修复

- **文件**：`UserItemController.java`
- **问题**：直接从 `Map` 取值，无校验
- **修复**：新增 `UserItemSaveRequest` DTO + `@Valid`

### Q-7 `FetchConsumer.autoTag()` 异常静默吞没 — 🚧 待修复

- **文件**：`FetchConsumer.java`
- **问题**：`catch (Exception ignored) {}` 完全忽略异常
- **建议**：至少记录 WARN 日志

### Q-8 前端纯 JavaScript — 🚧 待修复

- **问题**：无 TypeScript、无 ESLint/Prettier 配置
- **建议**：逐步迁移 TypeScript + 配置 ESLint + Prettier

## 总结

**Verdict: APPROVE with suggestions**

Windows 版本是对 macOS 原始版的全面升级：
- ✅ 两项 Critical 运行时缺陷已修复
- ✅ 6/7 项安全问题已修复
