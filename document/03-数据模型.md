# 数据模型

---

## 数据库表（6 张）

### items — 核心内容表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK AUTO_INCREMENT | 自增主键 |
| `type` | VARCHAR(20) | 品类：game / movie / anime / boardgame / model / book / music / digital / coffee / offline |
| `title` | VARCHAR(255) | 标题 |
| `slug` | VARCHAR(255) UNIQUE | URL 友好标识 |
| `cover_url` | VARCHAR(512) | 横版封面图 |
| `wide_cover_url` | VARCHAR(512) | 宽幅封面（备选） |
| `poster_url` | VARCHAR(512) | 竖版海报图（管理员上传） |
| `media_url` | VARCHAR(512) | 视频/媒体 URL |
| `external_link` | VARCHAR(512) | 外部商店链接 |
| `description` | TEXT | 描述 |
| `info_json` | JSON | 品类特定结构化数据（见下方） |
| `external_id` | VARCHAR(100) | Steam AppID / TMDB ID 等 |
| `source` | VARCHAR(30) | 数据来源：manual / steam / tmdb / itch 等 |
| `status` | TINYINT DEFAULT 1 | 1=online（已发布）, 0=offline（待审核） |
| `carousel_order` | INT NULL | NULL=不在轮播中, 0,1,2...=轮播排序 |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 更新时间（ON UPDATE CURRENT_TIMESTAMP） |

> **注意**：`status` 使用 TINYINT 而非 VARCHAR。后端 `AdminController` 中 `pending` 接口查 `status=0`，`approve` 设为 `1`。

### users — 用户表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | |
| `username` | VARCHAR(50) UNIQUE | |
| `email` | VARCHAR(100) UNIQUE | |
| `password_hash` | VARCHAR(255) | BCrypt 加密 |
| `avatar` | VARCHAR(255) | 头像 URL |
| `role` | VARCHAR(20) DEFAULT 'USER' | USER / ADMIN |

### user_items — 收藏关联

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | |
| `user_id` | BIGINT FK → users.id | |
| `item_id` | BIGINT FK → items.id | |
| `status` | VARCHAR(20) | want_to_play / played / loved / dropped |
| `played_duration` | INT | 游玩时长（秒） |
| `note` | TEXT | 备注 |

### tags — 标签表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | |
| `name` | VARCHAR(50) UNIQUE | |

### item_tag_mapping — 多对多关联

| 字段 | 类型 | 说明 |
|------|------|------|
| `item_id` | BIGINT | → items.id |
| `tag_id` | BIGINT | → tags.id |
| | PRIMARY KEY | (item_id, tag_id) |

### reviews — 评论表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK | |
| `user_id` | BIGINT | → users.id |
| `item_id` | BIGINT | → items.id |
| `rating` | TINYINT | 评分 |
| `comment` | TEXT | 评论内容 |
| `created_at` | DATETIME | |

## info_json 结构

`info_json` 使用 MySQL JSON 类型，新增品类无需改表。以下为各品类代表性字段：

### 游戏 (type=game)

```json
{
  "developer": "Supergiant Games",
  "genre": "动作, Rogue-like",
  "platform": "PC, PS5, Xbox",
  "demo_available": true,
  "demo_url": "steam://install/2716740",
  "videos": {
    "youtube": "https://www.youtube.com/embed/...",
    "bilibili": "//player.bilibili.com/player.html?bvid=BV1..."
  }
}
```

### 电影 (type=movie)

```json
{
```
