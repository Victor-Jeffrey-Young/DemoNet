### items — 核心内容表

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK AUTO_INCREMENT | 自增主键 |
| `type` | VARCHAR(20) NOT NULL | 品类：game / movie / anime / boardgame / model / book / music / digital / coffee / offline |
| `title` | VARCHAR(255) NOT NULL | 标题 |
| `slug` | VARCHAR(255) UNIQUE | URL 友好标识（唯一） |
| `cover_url` | VARCHAR(512) | 横版封面图 |
| `wide_cover_url` | VARCHAR(512) | 宽幅封面（备选） |
| `poster_url` | VARCHAR(512) | 竖版海报图（SteamGridDB / 管理员上传） |
| `media_url` | VARCHAR(512) | 视频/媒体 URL |
| `external_link` | VARCHAR(512) | 外部商店链接 |
| `description` | TEXT | 描述 |
| `info_json` | JSON | 品类特定结构化数据（见下方） |
| `external_id` | VARCHAR(100) | Steam AppID / TMDB ID 等 |
| `source` | VARCHAR(30) DEFAULT 'manual' | 数据来源：manual / steam / tmdb / itch 等 |
| `status` | TINYINT DEFAULT 1 | 1=online（已发布）, 0=offline（待审核） |
| `carousel_order` | INT NULL | NULL=不在轮播中, 0,1,2...=轮播排序 |
| `recommendations` | INT DEFAULT 0 | Steam 好评数，用于热度排序 |
| `hot_boost` | INT DEFAULT 0 | 管理员手动热度加权（新游戏自动设 200） |
| `created_at` | DATETIME DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**索引**：`idx_type(type)`, `idx_slug(slug)`

**热度排序公式**：
```sql
热度分 = (LN(recommendations + 1) * 100 + hot_boost + recent_reviews * 500)
         / GREATEST(DATEDIFF(NOW(), created_at), 1)
```

> **重要**：`status` 使用 TINYINT。`pending` 接口查 `status=0`，`approve` 设为 `1`。

### curation_collections — 场景策展集合

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | BIGINT PK AUTO_INCREMENT | 自增主键 |
| `slug` | VARCHAR(100) NOT NULL UNIQUE | 可分享场景页的稳定标识 |
| `title` | VARCHAR(120) NOT NULL | 面向用户的场景标题 |
| `description` | VARCHAR(500) NOT NULL | 场景承诺与适用说明 |
| `cover_url` | VARCHAR(512) NULL | 场景卡封面 |
| `constraints_json` | JSON NULL | 人数、时长、地点、预算等场景约束；结构由策展契约约束 |
| `visible` | TINYINT NOT NULL DEFAULT 0 | 1=公开，0=草稿/隐藏；公开接口只读取 1 |
| `display_order` | INT NOT NULL DEFAULT 0 | 首页场景轨道排序（升序） |
| `created_at` | DATETIME | 创建时间 |
| `updated_at` | DATETIME | 最近更新时间 |

**索引**：`UNIQUE(slug)`、`idx_curation_collections_visible_order(visible, display_order)`。

### curation_collection_items — 场景与内容关联

| 字段 | 类型 | 说明 |
|------|------|------|
| `collection_id` | BIGINT FK → curation_collections.id | 所属场景；删除集合时级联删除关联 |
| `item_id` | BIGINT FK → items.id | 入选内容；一项内容可加入多个场景 |
| `display_order` | INT NOT NULL DEFAULT 0 | 场景页内排序（升序） |
| `reason` | VARCHAR(300) NOT NULL | 面向用户的入选理由 |
| `editor_note` | TEXT NULL | 仅后台编辑备注，公开 API 不返回 |

**主键**：`(collection_id, item_id)`，防止同一内容在同一场景重复入选。公开查询同时要求集合 `visible=1` 与内容 `items.status=1`，因此隐藏场景和下线内容不会泄露到前台。

> 迁移来源：`db/migration/V4__Add_curation_collections.sql`。场景集合独立于 `hot_boost` 和 `carousel_order`，不改变热门趋势或既有编辑精选的含义。

## info_json 结构

`info_json` 使用 MySQL JSON 类型，新增品类无需改表。以下为各品类代表性字段：

### 跨品类体验契约（`info_json.experience`）

场景筛选和详情页的“体验条件”只读取这一稳定命名空间；品类编辑器中的既有字段继续保留，按下表兼容读取，**未知值宁可省略，不得臆造**。

```json
{
  "experience": {
    "actions": [
      {
        "type": "demo",
        "label": "试玩 Demo",
        "url": "https://example.com/demo",
        "source": "Steam 官方商店",
        "official": true,
        "checked_at": "2026-07-13"
      }
    ],
    "time_minutes": { "min": 15, "max": 30 },
    "players": { "min": 1, "max": 1 },
    "budget": { "currency": "CNY", "min": 0, "max": 0, "label": "免费" },
    "location_mode": "online",
    "requirements": ["可联网的电脑"],
    "source_checked_at": "2026-07-13"
  }
}
```

| 字段 | 规则 |
|------|------|
| `actions` | 可选数组。每项必须有 `type`、`label`、`url`、`source`、`official`、`checked_at`；`checked_at` 为 ISO 日期。`url` 必须是可公开打开的 HTTP(S) 链接。 |
| `actions[].type` | `demo` / `benchmark` / `trailer` / `listen` / `read` / `rules` / `reservation` / `purchase` / `external`。只有前七类可进入后续“现在可以体验”入口；`purchase`、`external` 只作为普通外链，不伪装成即时体验。 |
| `time_minutes`、`players` | 可选对象，`min`、`max` 为非负整数，存在时 `min ≤ max`。 |
| `budget` | 可选对象；金额以 `currency` + `min`/`max` 表达，无法可靠量化时只填写 `label`。 |
| `location_mode` | `online` / `offline` / `either`；不确定时省略。 |
| `requirements` | 可选字符串数组，例如设备、账号、材料或预约要求。 |
| `source_checked_at` | 整个体验信息最近人工核验日期；单个行动以 `actions[].checked_at` 为准。 |

兼容读取顺序：优先 `experience`，缺失时再读取旧字段；不会在迁移期批量改写已有 `info_json`。

| 旧字段 | 兼容为的行动 | 说明 |
|------|------|------|
| 游戏 `demo_url` | `demo` | 已有试玩链接才显示 |
| 游戏 `benchmark_url` | `benchmark` | 仅表示性能测试工具 |
| 电影/动漫 `trailer` 或视频链接 | `trailer` | 是预览，不等于完整观看 |
| 书籍 `reader_url` | `read` | 仅在确为试读/阅读入口时使用 |
| 音乐 `preview_url` | `listen` | 仅在确为试听入口时使用 |
| 桌游规则预览字段 | `rules` | 需要可打开的规则页或文件链接 |
| `external_link` | `external` | 默认普通外链；除非人工补充明确的 `experience.actions`，不纳入即时体验推荐 |

### 场景约束契约（`curation_collections.constraints_json`）

场景集合使用独立 JSON，表达“这个场景适合什么条件”，不复制具体内容的体验事实：

```json
{
  "people": { "min": 1, "max": 1 },
  "time_minutes": { "max": 30 },
  "location_modes": ["online"],
  "budget": { "label": "免费或低门槛" },
  "requirements": ["可联网设备"],
  "promise": "不需要提前准备，打开即可开始"
}
```

`people` 与 `time_minutes` 只允许非负整数范围且 `min ≤ max`；`location_modes` 只允许 `online`、`offline`、`either`；`requirements` 是场景层面的共通前提；`promise` 用一句话说明为何此场景值得打开。首批固定使用以下 slug，具体入选条目在来源、核验日期和理由齐备前保持隐藏：

| slug | 场景 |
|------|------|
| `solo-quick-start` | 一个人，30 分钟内，在线开始 |
| `friends-night` | 3–5 人的宿舍/朋友夜 |
| `easy-date` | 两个人的轻松约会 |
