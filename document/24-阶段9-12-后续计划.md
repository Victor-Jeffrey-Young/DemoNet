## 阶段 9：质量加固（Bug 修复 + 安全 + 架构优化）

> **关键词**：从"能用"到"可靠"

### 安全隐患（最高优先）

| # | 问题 | 位置 | 方案 |
|---|------|------|------|
| 9.1 | SQL 拼接注入风险 | `ItemService` | 改用 `JdbcTemplate` `?` 占位符或 MyBatis-Plus `LambdaQueryWrapper` |
| 9.2 | Admin 路由无前端守卫 | `router/index.js` | `/admin` 加 `meta: { requiresAuth: true, requiresAdmin: true }` + `beforeEach` 校验 |
| 9.3 | Admin 接口无后端角色校验 | `SecurityConfig` | `/api/admin/**` 加 `.hasRole("ADMIN")` |
| 9.4 | 敏感信息硬编码 | `docker-compose.yml`, `application.yml` | 改用 `${ENV_VAR}` 环境变量 + `.env` 文件 |

### 前端 Bug

| # | 问题 | 方案 |
|---|------|------|
| 9.5 | 401 重定向失效 | `window.location.hash` → `router.push('/login')` |
| 9.6 | Tailwind 动态类名不生效 | `types.js` 存储完整类名（`bgColor: 'bg-emerald-600'`）或 Tailwind safelist |
| 9.7 | Element Plus 全量引入占 ~1MB | 移除或改为按需引入（`unplugin-vue-components`） |

### 后端架构

| # | 问题 | 方案 |
|---|------|------|
| 9.8 | 无全局异常处理 | `GlobalExceptionHandler` + `BusinessException`，统一返回 `{ code, message }` |
| 9.9 | 无输入校验 | DTO 加 `@NotBlank/@Email/@Size`，Controller 加 `@Valid` |
| 9.10 | Entity 暴露敏感字段 | `passwordHash` 加 `@JsonIgnore`，或引入 DTO 层 |
| 9.11 | Entity/DB 不同步 | 引入 Flyway 管理 schema 迁移 |

### 前端架构

| # | 问题 | 方案 |
|---|------|------|
| 9.12 | 类型元数据 4 处重复 | 所有组件统一从 `constants/types.js` 导入 `getMeta()` |
| 9.13 | 无数据缓存 | Pinia `stores/item.js` 统一管理，跨组件复用 |
| 9.14 | 加载态简陋 | 骨架屏组件替代"加载中..."纯文字 |
| 9.15 | 空 catch 静默吞错 | 全局错误 Toast 提示，关键操作日志输出 |

## 阶段 10：用户互动层

> **关键词**：从"一个人的收藏夹"到"社区的试玩派对"

| # | 功能 | 说明 |
|---|------|------|
| 10.1 | 评论系统 | `reviews` 表已建，补 `ReviewController` + `ReviewSection.vue`，支持评分 + 文字评论 + 时间线 |
| 10.2 | 用户主页增强 | 收藏统计（各品类数量/状态分布）、评论历史、个人标签云 |
| 10.3 | 评论点赞 | `review_likes` 表，点赞数排序，帮用户发现有价值的评价 |
| 10.4 | 分享卡片 | `<meta og:title/og:image/og:description>` + 动态生成分享图，微信/朋友圈/Twitter 预览 |
| 10.5 | 通知系统 | 评论被回复、收藏内容更新、管理员推荐通知 |

## 阶段 11：发现与推荐

## 阶段 12：运营与规模化

> **关键词**：从"能跑"到"稳定跑、高效跑"

| # | 功能 | 说明 |
|---|------|------|
| 12.1 | CI/CD | GitHub Actions 自动构建 + 测试 + 部署到云服务器 |
