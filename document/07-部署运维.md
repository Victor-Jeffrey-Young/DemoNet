> Docker 部署 + 测试策略 + 上线清单

## 本地开发

### 环境准备

```bash
# 首次：生成 .env 文件
bash scripts/setup-env.sh

# 启动基础设施
docker compose up -d

# 后端 (:8080)
cd backend && mvn clean spring-boot:run

# 前端 (:5173)
cd frontend && npm run dev
```

### docker-compose.yml（开发）

| 服务 | 镜像 | 端口 |
|------|------|------|
| MySQL | `mysql:8.0` | 3306 |
| Redis | `redis:7-alpine` | 6379 |
| RabbitMQ | `rabbitmq:3.12-management` | 5672 / 15672 |

## 生产部署

### docker-compose.prod.yml（5 服务）

| 服务 | 说明 |
|------|------|
| `db` | MariaDB 11（替代 MySQL 8.0，生产更稳定） |
| `redis` | Redis 7-alpine |
| `rabbitmq` | RabbitMQ 3-management-alpine |
| `backend` | Spring Boot JAR → `backend/Dockerfile` |
| `frontend` | Nginx 静态文件 → `frontend/Dockerfile` + `nginx.conf` |

### 构建与启动

```bash
# 首次：生成 .env（填入生产密码）
bash scripts/setup-env.sh

# 构建 + 启动全部服务
docker compose -f docker-compose.prod.yml up -d --build
```

### Nginx 配置要点

- 前端静态文件由 Nginx 直接 serve
- `/api/` 路径反向代理到 `backend:8080`
- `/uploads/` 路径由后端 serve（或 Nginx 静态目录映射）
- 建议配置 gzip 压缩 + 静态资源缓存头

### 回滚方案

```bash
# 停止当前服务
docker compose -f docker-compose.prod.yml down

# 恢复数据库（如有备份）
# mysql -u root -p demonet < backup.sql

# 启动上一版本镜像
docker compose -f docker-compose.prod.yml up -d
```

## 环境变量 (.env)

参见 `.env.example`，主要变量：

| 变量 | 说明 |
|------|------|
| `MYSQL_ROOT_PASSWORD` | 数据库 root 密码 |
| `MYSQL_DATABASE` | 数据库名 |
| `REDIS_PASSWORD` | Redis 密码 |
| `RABBITMQ_DEFAULT_USER/PASS` | RabbitMQ 凭证 |
| `JWT_SECRET` | JWT 签名密钥 |
| `STEAM_API_KEY` | Steam API 密钥 |
| `TMDB_API_KEY` | TMDB API 密钥 |
| `IGDB_CLIENT_ID/SECRET` | Twitch IGDB OAuth |
| `APP_UPLOAD_DIR` | 上传文件路径 |
