## 基础设施（阶段 0-5）

### 1. mvnw 缺失
- **现象**：项目中无 `mvnw` 脚本
- **解决**：直接用全局 `mvn` 命令

### 2. MyBatis-Plus 依赖版本
- **现象**：启动报错，找不到 `MybatisPlusInterceptor`
- **解决**：使用 `mybatis-plus-spring-boot4-starter:3.5.16`

### 3-4. 分页 total=0
- **现象**：分页返回 total=0，但数据正常
- **根因**：缺少 SQL 解析依赖
- **解决**：添加 `mybatis-plus-jsqlparser:3.5.16`

### 5. JDBC 中文编码
- **解决**：`characterEncoding=UTF-8` 配置

### 6. 中文双重编码
- **现象**：Python 脚本插入中文后出现双重编码
- **解决**：Python 使用 `mysql-connector-python` 直连，设置 charset

### 7. VSCode ClassNotFoundException
- **现象**：IDE 运行报类未找到
- **解决**：`mvn clean` 清除缓存后重试

### 8. docker-compose version
- **现象**：新版 Docker Compose 报 `version` 已弃用
- **解决**：移除 `docker-compose.yml` 中的 `version` 行

### 9. Map.of 参数限制
- **现象**：超过 10 个条目编译报错
- **解决**：改用 `Map.ofEntries()`

### 10. 收藏下拉菜单消失
- **现象**：CSS `group-hover` 在快速移动鼠标时丢失状态
- **解决**：JS `mouseenter/mouseleave` 事件替代

### 11. Profile 页面无标题
- **现象**：收藏列表只显示 ID，不显示标题
- **解决**：`UserItemService` 使用 JOIN 查询 items 表返回 title/slug

### 12. seedTags() 空异常
- **现象**：查询空表报 NPE
- **解决**：`queryForList` 替代 `queryForObject`

### 13. 详情页 overflow-hidden 裁剪下拉
- **现象**：收藏下拉菜单被卡片 overflow-hidden 裁剪
- **解决**：下拉菜单移至图片区域外部

### 17. @Value 缺失 import
- **现象**：SteamService 编译报错
- **解决**：添加 `import org.springframework.beans.factory.annotation.Value`

### 18. 竖版卡片缺图

## 管理后台 / DB 同步

### 29. poster_url 列缺失导致整站瘫痪

- **时间**：2026-05-28
- **现象**：编译成功但所有页面报 500，日志显示 `Unknown column 'poster_url' in 'field list'`。影响所有涉及 `items` 表的查询（首页、列表、详情、搜索全挂）。
- **触发动作**：新增管理员界面前后端代码时，在 `Item.java` Entity 和 `schema.sql` 中新增了 `posterUrl` / `poster_url` 字段，但未对已存在的 MySQL 表执行 `ALTER TABLE`。
- **解决**：`docker exec demonet-mysql mysql -uroot -pchangeme demonet -e "ALTER TABLE items ADD COLUMN poster_url VARCHAR(512) AFTER media_url;"`

### 30. users 表 role 列缺失导致注册/登录失败

- **时间**：2026-05-28
- **现象**：注册和登录接口报 `Unknown column 'role' in 'field list'`，`User.java` 有 `private String role` 字段但 MySQL `users` 表无对应列。
- **触发动作**：与 #29 同一批改动，新增管理员角色字段但未迁移数据库。
- **解决**：

  ```bash
  docker exec demonet-mysql mysql -uroot -pchangeme demonet \
    -e "ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'USER' AFTER password_hash;"
  docker exec demonet-mysql mysql -uroot -pchangeme demonet \
    -e "UPDATE users SET role = 'ADMIN' WHERE username = 'admin';"
  ```

### 结论：#29 + #30

Entity 字段与数据库表不同步是当前项目的系统性问题。**每次在 Entity 中新增字段，必须同步执行对应的 ALTER TABLE。** 建议后续引入 Flyway/Liquibase 实现自动化迁移。

### 33. Safari 表格隔行模糊（一行清晰一行模糊）

- **时间**：2026-05-28
- **现象**：Safari 中 `el-table` 隔行文字渲染不一致——一行锐利、下一行模糊
- **根因**：`el-table` 的 `stripe` 属性对隔行添加 `el-table__row--striped` 类，在 Safari 中触发独立 GPU 合成层。不同合成层的子像素抗锯齿算法不一致，导致视觉模糊交替。卡片视图的 `backdrop-blur` 同理。
- **解决**：
  - 三张表全部移除 `stripe` 属性（AdminItemList / AdminTagManager / AdminFetchPanel）
  - 移除卡片品类徽章的 `backdrop-blur`
  - `Admin.vue` 全局 `.admin-tabs` 强制 `-webkit-font-smoothing: antialiased`，覆盖所有 `tr/td`

### 34. Select 筛选器点击前白底灰字

- **时间**：2026-05-28
- **现象**：筛选器未点击时显示白色背景+灰色文字（Element Plus 默认浅色主题）；点击后下拉列表配色正确（暗色）
- **根因**：Element Plus `el-select` 通过 CSS 变量 `--el-fill-color-blank`（默认白色）控制输入框背景，非 `background-color`。`.admin-tabs .el-input__wrapper` 的 `background-color` 被 CSS 变量覆盖。同时下拉面板 teleport 到 `<body>` 外，`.admin-tabs` 作用域 CSS 失效。
- **解决**：
  - 为 `.admin-tabs .el-select` 覆盖 4 个 CSS 变量：`--el-fill-color-blank`/`--el-border-color`/`--el-text-color-regular`/`--el-text-color-placeholder`
  - `!important` 加固 `.el-input__wrapper` 背景
  - 所有 `el-select` 加 `:teleported="false"` + `popper-class="admin-select-drop"` 兜底
  - 新增全局 `.admin-select-drop` 暗色规则（底 `#1f2937`、字 `#e5e7eb`、hover `#374151`、选中 `#60a5fa`、箭头同步）

### 35. 标签管理页面不显示标签

- **时间**：2026-05-28
- **现象**：标签管理表格为空白（共 0 个），数据库有 78 条标签
- **根因**：`SecurityConfig` 新增 `hasRole("ADMIN")` 保护 `/api/admin/**`，但 admin 的 JWT token 在 `role` 字段加入 before 生成，不含 `role` 声明。`JwtAuthFilter` 取不到 role 默认赋 `ROLE_USER`，所有 admin API 返回 403。前端 catch 块只 `console.error`，未提示用户。
- **解决**：
  - **临时**：admin 退出重新登录，新 token 含 `role: "ADMIN"`
  - **长期**：前端 403 时弹出提示"请重新登录以获取管理员权限"

### 36. 添加重复标签时 500 报错

- **时间**：2026-05-28
- **现象**：添加已存在的标签（如"潮汕"）时 MySQL 抛 `Duplicate entry for key 'tags.name'` → HTTP 500 + Java 堆栈
- **根因**：`TagService.create()` 未做去重检查，直接 `insert`。MySQL UNIQUE 约束兜底抛异常，前端收到原始堆栈错误。
- **解决**：
  - `TagService.create()` 插入前 `SELECT COUNT(*)` 查重，已存在抛出 `RuntimeException("标签「潮汕」已存在")`
  - 新建 `GlobalExceptionHandler`（`@RestControllerAdvice`）将 `RuntimeException` → HTTP 400 + `{"error": "消息"}`，不再泄露堆栈
  - 前端改为从 `e.response.data.error` 读取清晰错误

### 37. 标签搜索跨页失效

- **时间**：2026-05-28
- **现象**：标签"潮汕"在第 4 页，在第 1 页搜索框输入"潮汕"显示 No data，翻到第 4 页才可见
- **根因**：搜索是纯客户端在 `tags` 数组（仅当前页 20 条）上做 `filter()`，目标标签不在当前页就搜不到
- **解决**：
  - 后端 `TagService.listPaginated()` + `keyword` 参数，`LIKE` 全表匹配后分页返回
  - 前端去掉 `filteredTags` 客户端过滤，搜索框 300ms 防抖直接调后端 `/api/admin/tags/paged?keyword=xxx`
  - 搜索结果多于一页时分页器保持可用

### 38. TMDB 抓取后待审核队列不刷新

- **时间**：2026-05-28
- **现象**：提交 TMDB/Steam 抓取任务后，待审核列表不变化，需手动切 tab 或刷新页面才能看到新条目
- **根因**：前端 `handleTMDBFetch()` 只发 RabbitMQ 消息和显示"任务已提交"，从未调用 `loadPending()` 刷新队列
- **解决**：
  - 提交后 `setTimeout(loadPending, 2000)` 延迟 2 秒自动刷新
  - 队列标题旁加「刷新」按钮供手动触发

### 39. TMDB 中文查询 URL 编码缺失

- **时间**：2026-05-28
- **根因**：`TMDBService.searchMovies()` 拼 URI 时 `query` 参数含中文（如"奥本海默"），未做 `URLEncoder.encode()`，TMDB API 可能返回空或报错
- **解决**：`URLEncoder.encode(query, StandardCharsets.UTF_8)` 编码后拼接 URI

### 40. 电影轮播"观看预告"按钮无响应

### 41. TMDB 抓取遇到重复 slug 时整批中断

- **时间**：2026-05-28
- **现象**：抓取已知电影时 RabbitMQ 报 `ListenerExecutionFailedException`，`Duplicate entry 'tmdb-xxx' for key 'items.slug'`，后续未重复条目也一并丢失
- **根因**：`FetchConsumer` 对搜索结果逐条 `createItem()`，遇 MySQL UNIQUE 约束立刻抛异常退出循环
- **解决**：Steam 和 TMDB 消费者均改为 `try-catch DuplicateKeyException`，重复 → `skipped++` 继续下一条；日志输出 `"{} saved, {} skipped"`

### 42. WebP 图片上传失败

### 49. 轮播管理重构

- **时间**：2026-05-28
- **背景**：原精选（★ + `is_featured`）散落在内容列表里，无排序，无预览，无独立管理入口
- **数据层**：`is_featured` → `carousel_order INT NULL`（NULL=不在轮播，0,1,2...=展示顺序，自动迁移旧数据）
- **后端 API**（3 个新端点）：
  - `GET /api/admin/carousel/{type}` — 获取当前轮播序列
  - `POST /api/admin/carousel/{type}` — 全量保存 `{ itemIds: [3,1,5] }`（清空→重新赋值 0,1,2...）

### 60. PDF 被 X-Frame-Options 阻止

- **时间**：2026-05-28
- **现象**：Safari 中 `<embed>` 加载 PDF 报 `Refused to display ... in a frame because it set 'X-Frame-Options' to 'DENY'`
- **根因**：Spring Security 默认 `X-Frame-Options: DENY` 响应头，阻止浏览器在任何 frame/embed 中加载本站资源
- **解决**：`SecurityConfig` 加 `.headers(headers -> headers.frameOptions(frame -> frame.sameOrigin()))`，从 `DENY` 改为 `SAMEORIGIN`

### 63. CoverFlow 重构历程

- **时间**：2026-05-28
- **演进**：
  - v1 — 原生滚动 `scroll-snap` + JS 计算 → Safari `scrollWidth` 算错 + `@scrollend` 不支持 → 失败
  - v2 — 纯 JS `active` 索引 + CSS transition → 能动但不像 CoverFlow（无重叠、无圆弧）→ 用户不满意
  - v3 — 回归原生滚动 + `CARD_PITCH=60` 硬编码 + `@scroll` 防抖 → `scrollBy` 不生效 → 失败
  - v4 — 回到纯 JS `active` → 视觉不够 → 用户不满意
  - v5 — 三方案参考（DeepSeek 圆弧几何 + Gemini 最短路径 + MIMO 阈值拖动）→ **最终方案**
- **最终架构**：
  - 圆弧几何：`translate3d(Rsinθ, 0, R(cosθ-1)) rotateY(-θ)`，`θ=STEP×offset`
  - 循环：`shortestPath` 最短绕行距离（`diff > N/2 → diff-N`）
  - 拖动：`dragOffset/gap > 0.35` 阈值判定，`transition:none` 跟手 + `cubic-bezier(0.2,0.9,0.3,1)` 弹性归位
  - GPU 修复：`visibilitychange` → `display:none` + `offsetHeight` reflow 重建图层
- **倒影**：最终采用 DOM-based `scaleY(-1)` + `mask-image` 渐变遮罩（`-webkit-box-reflect` 在 `position:absolute` 嵌套栈中完全不可见）
- **视觉效果**：360px 封面 + 650px 弧半径 + 900px 透视 + 120px 倒影，占 `100vh-4rem` 全屏

### 64. iTunes 抓取失败：text/javascript Content-Type 不兼容

- **时间**：2026-05-29
- **现象**：iTunes 抓取日志报 `Could not extract response: no suitable HttpMessageConverter found for response type [interface java.util.Map] and content type [text/javascript;charset=utf-8]`，0 saved。
- **根因**：iTunes Search API 返回 `Content-Type: text/javascript;charset=utf-8`，Spring `RestClient` 默认 `MappingJackson2HttpMessageConverter` 只认 `application/json`，不识别 `text/javascript`。
- **解决**：`ItunesService` 构造函数中自定义 `MappingJackson2HttpMessageConverter`，`supportedMediaTypes` 增加 `new MediaType("text", "javascript")`，通过 `RestClient.builder().messageConverters()` 注入。
