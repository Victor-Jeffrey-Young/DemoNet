# Context

DemoNet is preparing for a production launch. The current codebase has accumulated technical debt and security risks during rapid iteration (e.g., direct JDBC queries in controllers, missing schema migrations, missing MIME validation for uploads, and a monolithic `Detail.vue` view). The goal is to harden security, unify the database schema management, clean up architectural debt, and ensure the deployment configuration is fully production-ready without changing the existing product features.

# Approach

## 1. Security Hardening (Independent)
Fix vulnerabilities in file uploads, category updates, and password resets.
- **MIME Type Validation**:
  - In `backend/src/main/java/com/example/demonet/controller/AdminController.java` -> `uploadImage()`.
  - Extract the content type using `file.getContentType()`.
  - Reject uploads if the content type is not in the allowed list: `image/jpeg`, `image/png`, `image/webp`, `image/gif`. For `type = "reader"`, also allow `application/pdf` and `application/epub+zip`.
- **Category Type Whitelisting**:
  - In `AdminController.java` -> `updateCategorySettings()`.
  - Validate that each `type` is one of: `game`, `movie`, `anime`, `boardgame`, `model`, `book`, `music`, `digital`, `coffee`, `offline`. Discard invalid entries to prevent SQL injection via `REPLACE INTO`.
- **Password Reset Privacy**:
  - In `AdminController.java` -> `resetPassword()`.
  - Instead of returning `Map.of("password", newPassword)`, return `Map.of("message", "密码已重置为: " + newPassword + " (请将此密码复制发送给用户，窗口关闭后将无法再次查看)")` to make the one-time visibility explicit for the admin UI.

## 2. Frontend Bug Fixes & Optimization (Independent)
Resolve console warnings and bundle size issues.
- **AdminDashboard TypeIcon Fix**:
  - In `frontend/src/components/admin/AdminDashboard.vue`, add `import TypeIcon from '../TypeIcon.vue'` to the `<script setup>` to fix the missing component warning.
- **GameEditorFields Fragment Warning Fix**:
  - In `frontend/src/components/admin/GameEditorFields.vue`, add `defineOptions({ inheritAttrs: false })` in `<script setup>` to prevent attribute fallthrough warnings on the fragment root.
- **ECharts Lazy Loading**:
  - In `frontend/src/views/Detail.vue`, remove the top-level `import * as echarts from "echarts/dist/echarts.esm.js"`.
  - Inside `async function renderRadarChart()`, dynamically import it: `const echarts = await import('echarts/dist/echarts.esm.js');`. This prevents ECharts from bloat-loading on non-coffee detail pages.
- **Cleanup**:
  - Delete `frontend/src/components/HelloWorld.vue`.
  - Append `fix_images.py` and `demonet-coffee-plan.md` to `.gitignore`.

## 3. Database Schema & Flyway Migration (Requires Step 1)
Consolidate fragmented schema definitions into Flyway migrations to ensure environment reproducibility.
- **Add Flyway Dependencies**:
  - In `backend/pom.xml`, add `flyway-core` and `flyway-mysql` dependencies.
- **Create Baseline Migration**:
  - Move the contents of `backend/src/main/resources/schema.sql` to `backend/src/main/resources/db/migration/V1__Baseline.sql`.
  - Add the missing `users.enabled` column: `ALTER TABLE users ADD COLUMN enabled TINYINT DEFAULT 1;` (wrapped in a safe check or just define it in the `CREATE TABLE users` if we are assuming a clean baseline). Wait, since we are setting up a baseline, merge ALL missing tables (`category_settings`, `invite_codes`, `app_settings`) and columns into `V1__Baseline.sql` as `CREATE TABLE IF NOT EXISTS` and proper definitions.
- **Remove Dynamic DDL**:
  - In `backend/src/main/java/com/example/demonet/service/DataImportService.java`, remove any `jdbcTemplate.update("CREATE TABLE IF NOT EXISTS ...")` calls for `invite_codes` or `app_settings` (these are implied to exist if we look at how DataImportService might be structured, or if they were added via manual SQL previously).

## 4. Architectural Cleanup (Requires Step 3)
Remove direct `JdbcTemplate` usage from Controllers to enforce proper MVC layers.
- **Create Missing Entities & Mappers**:
  - Create `AppSetting.java` and `AppSettingMapper.java`.
  - Create `InviteCode.java` and `InviteCodeMapper.java`.
  - Create `CategorySetting.java` and `CategorySettingMapper.java`.
- **Refactor AdminController**:
  - Inject the newly created Mappers/Services.
  - Replace `jdbcTemplate.queryForList` and `jdbcTemplate.update` in `getCategorySettings`, `updateCategorySettings`, `listInviteCodes`, `generateInviteCodes`, `getAllSettings`, `updateSetting`, `listUsers`, `toggleBan`, and `resetPassword` with MyBatis-Plus `selectList`, `update`, `insert`, etc.
- **Refactor AuthController**:
  - Move the `verifyTurnstile` and `verifyInviteCode` methods, along with their `AppSetting` lookups, into `AuthService.register()`. `AuthController.register()` should only handle the Redis rate limiting and then call `authService.register(req, turnstileToken, inviteCode)`.

## 5. Production Configuration
Ensure the production docker-compose is fully wired.
- **Update docker-compose.prod.yml**:
  - Under the `backend` service, add missing environment variables to match `application.yml` and `.env.example`:
    - `ADMIN_DEFAULT_PASSWORD: ${ADMIN_DEFAULT_PASSWORD:-changeme}`
    - `APP_STEAMGRIDDB_API_KEY: ${STEAMGRIDDB_API_KEY:-}`
    - `SPRING_REDIS_PASSWORD: ${REDIS_PASSWORD:-}`
    - `SPRING_RABBITMQ_USERNAME: ${RABBITMQ_USER:-admin}`

# Critical files & anchors

- `backend/src/main/java/com/example/demonet/controller/AdminController.java` : Target `uploadImage`, `resetPassword`, and all `jdbcTemplate` usages.

- `docker-compose.prod.yml` : Target the `backend` service `environment` block.

# Assumptions & contingencies

- **Flyway Baseline**: Assumes that existing production databases (if any) can be baselined (`spring.flyway.baseline-on-migrate=true`). If this fails, the deployer will need to manually run `flyway baseline` on the production database. We will add `spring.flyway.baseline-on-migrate: true` to `application.yml`.
- **ECharts Dynamic Import**: Assumes Vite correctly chunks the dynamic import for ECharts. If it fails to resolve at runtime, we will revert to static import but move it to a dedicated wrapper component `CoffeeRadarChart.vue` to isolate the bloat.
- **Turnstile Verification**: Assumes the Turnstile secret verification endpoint responds identically when moved to `AuthService`. The degraded network fallback logic (`log.warn`) must be preserved during the move.
