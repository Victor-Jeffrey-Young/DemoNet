# ADR-008: 安全架构 — 多层防护体系

## 状态
已接受（Accepted）

## 日期
2025-03-20

## 上下文

DemoNet 是一个公开的 Web 应用，需要防护以下安全威胁：

- **未授权访问**：管理员接口需要权限验证
- **暴力破解**：登录接口需要限流防护
- **SQL 注入**：用户输入可能包含恶意 SQL
- **XSS 攻击**：用户提交的评论、描述可能包含恶意脚本
- **文件上传漏洞**：管理员上传图片可能包含恶意文件
- **Token 泄露**：JWT Token 需要安全存储和传输

### 关键需求
- **认证**：用户登录、权限验证
- **授权**：管理员接口保护
- **限流**：防止暴力破解和 DoS 攻击
- **输入验证**：防止 SQL 注入、XSS 攻击
- **安全头部**：防止点击劫持、MIME 类型嗅探

### 可选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Spring Security + JWT + Redis 限流（当前方案）** | 成熟、灵活、多层防护 | 配置复杂 |
| **仅 JWT 认证** | 简单 | 无暴力破解防护 |
| **OAuth2 + 第三方登录** | 安全、用户友好 | 复杂度高、需要授权服务器 |
| **API Key 认证** | 简单、适合机器调用 | 不适合用户认证 |

## 决策

**采用多层安全防护体系**：

### 第一层：认证（Authentication）— JWT

```java
// JwtAuthFilter.java
@Component
public class JwtAuthFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        String token = resolveToken(request);
        if (token != null && jwtTokenProvider.validateToken(token)) {
            Long userId = jwtTokenProvider.getUserId(token);
            // 设置认证信息
            UsernamePasswordAuthenticationToken auth =
                new UsernamePasswordAuthenticationToken(userId, null, authorities);
            SecurityContextHolder.getContext().setAuthentication(auth);
        }
        filterChain.doFilter(request, response);
    }
}
```

**安全特性**：
- **HMAC-SHA256 签名**：防止 Token 被篡改
- **256-bit 密钥**：通过环境变量 `JWT_SECRET` 注入
- **24 小时过期**：减少 Token 泄露的风险窗口
- **Token 黑名单**：注销时写入 Redis，下次请求拒绝访问

### 第二层：授权（Authorization）— 基于角色的访问控制

```java
// SecurityConfig.java
.authorizeHttpRequests(auth -> auth
    // 公开接口
    .requestMatchers("/api/auth/login", "/api/auth/register").permitAll()
    .requestMatchers(HttpMethod.GET, "/api/items/**").permitAll()

    // 管理员接口
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    .requestMatchers(HttpMethod.POST, "/api/items/**").hasRole("ADMIN")
    .requestMatchers(HttpMethod.DELETE, "/api/tags/**").hasRole("ADMIN")
)
```

**角色设计**：
- **ROLE_USER**：普通用户，可浏览、评论、收藏
- **ROLE_ADMIN**：管理员，可 CRUD 物品、标签、场景

### 第三层：限流（Rate Limiting）— Redis 计数器

```java
@Component
public class RateLimitFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request,
                                    HttpServletResponse response,
                                    FilterChain filterChain) throws ServletException, IOException {
        String ip = request.getRemoteAddr();
        String key = "rate_limit:" + ip + ":" + request.getRequestURI();

        Long count = redisTemplate.opsForValue().increment(key);
        if (count == 1) {
            redisTemplate.expire(key, Duration.ofMinutes(1));
        }

        if (count > RATE_LIMIT_THRESHOLD) {
            response.setStatus(HttpStatus.TOO_MANY_REQUESTS.value());
            return;
        }
        filterChain.doFilter(request, response);
    }
}
```

**限流规则**：
- **登录接口**：5 次/分钟/IP
- **注册接口**：3 次/分钟/IP
- **其他接口**：60 次/分钟/IP（默认）

### 第四层：输入验证（Input Validation）

```java
// 使用 @Valid 注解验证请求参数
@PostMapping("/items")
public ApiResponse<Item> createItem(@Valid @RequestBody CreateItemRequest request) {
    // @Valid 会自动验证 request 对象的字段约束
}

// CreateItemRequest.java
public class CreateItemRequest {
    @NotBlank(message = "标题不能为空")
    @Size(max = 256, message = "标题不能超过 256 个字符")
    private String title;

    @NotNull(message = "品类不能为空")
    private String type;

    // getters and setters
}
```

**MyBatis-Plus 参数化查询**：
```java
// ✅ 安全：LambdaQueryWrapper 自动参数化
LambdaQueryWrapper<Item> wrapper = new LambdaQueryWrapper<>();
wrapper.eq(Item::getSlug, slug);  // 参数化查询，防止 SQL 注入

// ❌ 不安全：字符串拼接
String sql = "SELECT * FROM items WHERE slug = '" + slug + "'";  // SQL 注入风险
```

### 第五层：安全响应头

```java
// WebConfig.java
@Override
public void addCorsMappings(CorsRegistry registry) {
    registry.addMapping("/api/**")
        .allowedOrigins("http://localhost:5173")  // 仅允许前端域名
        .allowedMethods("GET", "POST", "PUT", "DELETE")
        .allowCredentials(true)
        .maxAge(3600);
}
```

**安全头部**：
- **CORS**：仅允许信任的域名访问
- **CSRF 禁用**：JWT 无状态，无需 CSRF 防护
- **X-Frame-Options**：防止点击劫持（`SAMEORIGIN`）

## 替代方案被拒绝的原因

### 仅 JWT 认证
- ❌ **无暴力破解防护**：攻击者可以无限次尝试登录
- ❌ **无输入验证**：容易受到 SQL 注入、XSS 攻击

### OAuth2 + 第三方登录
- ❌ **过度设计**：当前无第三方登录需求
- ❌ **复杂度高**：需要授权服务器（如 Keycloak、Auth0）

### 会话 + Cookie
- ❌ **CSRF 风险**：需要额外的 CSRF Token 防护
- ❌ **跨域复杂**：需要配置 `SameSite`、`Secure` 等 Cookie 属性
- ❌ **分布式部署复杂**：需要 Session 共享存储

## 后果

### ✅ 正面影响
- **多层防护**：认证 + 授权 + 限流 + 输入验证 + 安全头部
- **暴力破解防护**：Redis 限流防止密码暴力破解
- **SQL 注入防护**：MyBatis-Plus 参数化查询 + @Valid 验证
- **XSS 防护**：前端 Vue 3 自动转义 + 后端输入验证

### ⚠️ 负面影响
- **配置复杂**：多层安全配置需要仔细调试
- **性能开销**：每次请求都要验证 JWT + 限流检查（约 1-2ms）
- **维护成本**：需要定期更新依赖、审查安全配置

### 📋 待优化项
1. **Turnstile 验证码**：登录接口添加 Cloudflare Turnstile 验证码，防止机器人攻击
2. **IP 白名单**：管理员接口支持 IP 白名单
3. **密码强度要求**：强制要求密码包含大小写、数字、特殊字符
4. **审计日志**：记录所有管理员操作（谁、何时、做了什么）
