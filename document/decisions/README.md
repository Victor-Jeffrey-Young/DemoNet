## 🎯 何时写 ADR？

根据 [Documentation and ADRs Skill](../.claude/skills/documentation-and-adrs)，以下情况需要写 ADR：

- ✅ 选择框架、库或主要依赖
- ✅ 设计数据模型或数据库 schema
- ✅ 选择认证策略
- ✅ 决定 API 架构（REST vs GraphQL）
- ✅ 选择构建工具或部署平台
- ✅ **任何反转成本很高的决策**

---

## 📝 ADR 生命周期

```
PROPOSED（提议） → ACCEPTED（接受） → SUPERSEDED（被替代）/ DEPRECATED（废弃）
```

- **不要删除旧的 ADR** — 它们保存了历史上下文
- 当决策改变时，编写新的 ADR 引用并替代旧的
