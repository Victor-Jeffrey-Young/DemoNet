# ADR-004: 数据库选型 — MySQL 8.0

## 状态
已接受（Accepted）

## 日期
2025-01-15（项目初期）

## 上下文

DemoNet 需要关系型数据库存储以下数据：
- **items**（10 个品类的统一条目，约 1000+ 行）
- **users**（用户账号、角色）
- **reviews**（用户评论、评分）
- **tags**（标签系统）
- **curation_collections**（场景策展集合）
- **user_items**（用户收藏/待玩清单）

### 关键需求
- **关系型数据**：用户、物品、评论、标签之间存在明确关系
- **事务支持**：用户操作需要 ACID 保证（如创建评论、更新状态）
- **JSON 支持**：`info_json` 字段存储多态内容
- **全文搜索**：物品标题、描述的模糊搜索
- **运维成本**：小团队，需要托管服务或 Docker 部署

### 可选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **MySQL 8.0（当前方案）** | 成熟、生态丰富、团队熟悉 | 全文搜索能力弱于 PostgreSQL |
| **PostgreSQL** | 强大的 JSONB、全文搜索、复杂查询 | 团队不熟悉、托管服务选择少 |
| **MongoDB** | 灵活 Schema、天然支持 JSON | 放弃关系型数据库的 ACID 和 JOIN |
| **SQLite** | 零配置、嵌入式 | 不适合多用户并发写入、无托管服务 |

## 决策

**使用 MySQL 8.0（生产环境 MariaDB 11）作为主数据库**：

### 技术细节
- **版本**：开发/测试 MySQL 8.0，生产 MariaDB 11（兼容 MySQL 协议）
- **ORM**：MyBatis-Plus 3.5.16（灵活的 SQL 控制 + 分页插件）
- **连接池**：HikariCP（Spring Boot 4 默认，性能最优）
- **迁移工具**：Flyway（V1-V4 迁移脚本）
- **JSON 支持**：MySQL 8.0 的 `JSON` 数据类型 + `->` 操作符

### 数据库配置（application.yml）
```yaml
spring:
  datasource:
    url: jdbc:mysql://${MYSQL_HOST}:3306/demonet?useSSL=false&serverTimezone=Asia/Shanghai
    username: ${MYSQL_USER:root}
    password: ${MYSQL_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver

mybatis-plus:
  configuration:
    map-underscore-to-camel-case: true
```

### Flyway 迁移脚本
- **V1**：Baseline（创建 9 张核心表）
- **V2**：Indexes（添加 15+ 个索引）
- **V3**：Foreign Keys（添加外键 + 级联规则）
- **V4**：Scene 功能（新增 `curation_collections` + `curation_collection_items`）

## 替代方案被拒绝的原因

### PostgreSQL
- ❌ **团队不熟悉**：团队成员对 MySQL 更熟悉，学习成本高
- ❌ **托管服务选择少**：国内云服务商（阿里云、腾讯云）对 PostgreSQL 的托管支持不如 MySQL 成熟
- ❌ **迁移成本高**：现有代码、Docker 镜像、部署脚本都基于 MySQL

### MongoDB
- ❌ **放弃 ACID 事务**：用户评论、物品状态更新等操作需要事务保证
- ❌ **放弃 JOIN**：评论列表需要关联用户信息、物品信息，MongoDB 的 `$lookup` 性能不如 MySQL JOIN
- ❌ **放弃成熟生态**：MyBatis-Plus、Flyway 等工具对 MySQL 支持最好

### SQLite
- ❌ **并发写入限制**：SQLite 只支持单线程写入，不适合多用户并发场景
- ❌ **无托管服务**：生产环境无法使用 SQLite
- ❌ **功能缺失**：不支持 JSON 类型、全文搜索能力弱

## 后果

### ✅ 正面影响
- **团队熟悉**：MySQL 是团队最熟悉的数据库，降低开发和维护成本
- **生态丰富**：MyBatis-Plus、Flyway、DBeaver 等工具完美支持
- **托管服务成熟**：阿里云 RDS、腾讯云 CDB 都有成熟的 MySQL 托管服务
- **JSON 支持**：MySQL 8.0 的 `JSON` 类型足够满足 `info_json` 的存储需求

### ⚠️ 负面影响
- **全文搜索能力弱**：MySQL 8.0 的全文搜索不如 PostgreSQL 的 `tsvector` 强大
  - **缓解措施**：当前使用 `LIKE '%keyword%'` 模糊搜索，后续可引入 Elasticsearch
- **JSON 查询性能**：MySQL 对 JSON 内部字段的查询性能不如 PostgreSQL 的 JSONB
  - **缓解措施**：高频查询字段（如 `developer`、`author`）可提取为独立列

### 📋 优化方向
1. **MySQL JSON 索引**：对 `info_json` 的高频查询字段建索引（如 `developer`、`author`）
2. **引入 Elasticsearch**：为物品标题、描述、标签提供全文搜索能力
3. **读写分离**：引入 MySQL 主从复制，分离读请求和写请求
