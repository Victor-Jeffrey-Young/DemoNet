# ADR-005: 前端技术栈 — Vue 3 + Tailwind CSS 4 + Element Plus

## 状态
已接受（Accepted）

## 日期
2025-01-20

## 上下文

DemoNet 前端需要支持 10 个品类的**独立视觉语言**，同时保持代码的可维护性和开发效率。

### 关键需求
- **响应式设计**：支持桌面、平板、手机多端
- **组件复用**：10 个品类的 Hero/Card 组件需要高度定制化
- **状态管理**：用户认证、物品收藏、购物车等全局状态
- **路由管理**：多页面 + 权限控制（Admin 后台）
- **开发效率**：快速迭代、热重载、类型安全

### 可选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Vue 3 + Tailwind CSS 4（当前方案）** | 灵活、性能好、生态成熟 | 需要手动处理组件样式 |
| **React + Material-UI** | 组件库丰富、生态最大 | 学习曲线陡峭、JSX 不如 Vue template 直观 |
| **Vue 3 + Nuxt.js** | SSR/SSG、SEO 友好 | 复杂度高、项目初期不需要 SSR |
| **Next.js** | React 生态 + SSR | 同上，过度设计 |

## 决策

**使用 Vue 3 + Vite + Tailwind CSS 4 + Element Plus 技术栈**：

### 核心技术栈
| 技术 | 版本 | 用途 |
|------|------|------|
| **Vue 3** | 3.5+ | 响应式 UI 框架 |
| **Vite** | 8.0+ | 构建工具（比 Webpack 快 10-100 倍） |
| **Pinia** | 3.0+ | 状态管理（替代 Vuex） |
| **Vue Router 4** | 4.6+ | 路由管理 |
| **Tailwind CSS 4** | 4.3+ | 原子化 CSS 框架 |
| **Element Plus** | 2.14+ | UI 组件库（表单、弹窗、消息等） |
| **Axios** | 1.6+ | HTTP 客户端 |
| **ECharts** | 6.1+ | 数据可视化 |

### 设计系统
- **10 品类独立视觉语言**：每个品类有独立的配色、图标、Hero 设计
- **Tailwind CSS 4**：使用 CSS 变量 + `@theme` 实现动态主题切换
- **Element Plus**：用于通用组件（表单、弹窗、消息提示）

### 代码结构
```
frontend/src/
├── api/              # API 封装（按域划分）
├── components/       # 组件
│   ├── category/     # 10 个品类组件（Hero + Card）
│   ├── admin/        # 管理端编辑器
│   └── detail/       # 详情页子组件
├── constants/        # 常量（types.js 品类配置）
├── stores/           # Pinia 状态管理
├── views/            # 页面组件
└── router/           # 路由配置
```

## 替代方案被拒绝的原因

### React + Material-UI
- ❌ **学习曲线陡峭**：团队成员更熟悉 Vue 3
- ❌ **JSX 不如 Vue template 直观**：Vue 的 template 语法更易读
- ❌ **TypeScript 配置复杂**：Vue 3 的 `<script setup>` 语法糖更简洁

### Vue 3 + Nuxt.js
- ❌ **过度设计**：DemoNet 是内容聚合平台，SEO 需求不高（物品详情页 SEO 可有可无）
- ❌ **复杂度高**：SSR/SSG 增加了部署和维护成本
- ❌ **开发速度慢**：Nuxt.js 的约定优于配置对快速迭代不友好

### Next.js
- ❌ **React 生态**：团队更熟悉 Vue 3
- ❌ **SSR 过度设计**：同上，DemoNet 不需要 SSR

## 后果

### ✅ 正面影响
- **开发效率高**：Vite 热重载极快（毫秒级），开发体验好
- **灵活度高**：Tailwind CSS 4 的原子化 CSS 便于快速实现 10 个品类的独立视觉
- **生态成熟**：Vue 3 + Element Plus 的组件库丰富，遇到问题容易找到解决方案
- **类型安全**：Pinia + TypeScript（可选）提供良好的类型推断

### ⚠️ 负面影响
- **CSS 体积**：Tailwind CSS 的原子类导致 HTML 体积增大（但经过 PurgeCSS 优化后影响不大）
- **学习成本**：团队成员需要学习 Tailwind CSS 4 的新特性（如 `@theme`、CSS 变量）
- **Element Plus 定制化**：Element Plus 的样式定制不如纯 Tailwind CSS 灵活

### 📋 优化方向
1. **TypeScript 迁移**：当前使用 JavaScript，后续可迁移到 TypeScript
2. **组件库优化**：将 Element Plus 的表单组件封装为业务组件（如 `AdminItemForm`）
3. **性能优化**：图片懒加载、路由懒加载、组件按需加载
