# ADR-012: 数据库迁移策略 — Flyway

## 状态
已接受（Accepted）

## 日期
2025-04-15

## 上下文

DemoNet 的数据库 schema 会随着功能迭代不断演化：

- **V1**：初始 schema（items、users、reviews、tags、user_items、scenes、scene_items、curation_collections、curation_collection_items）
- **V2**：添加索引（加速查询）
- **V3**：添加外键（保证数据完整性）
- **V4**：新增 Scene 功能（curation_collections 表）

### 关键需求
- **版本管理**：每个 schema 变更都有对应的迁移脚本
- **自动化**：应用启动时自动执行未执行的迁移
- **回滚能力**：错误的迁移可以回滚
- **团队协作**：多人开发时 schema 变更不会冲突

### 可选方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Flyway（当前方案）** | 简单、成熟、SQL 优先 | 不支持自动回滚（需手动编写 undo 脚本） |
| **Liquibase** | 支持 XML/YAML/JSON/SQL、自动回滚 | 配置复杂、学习曲线陡 |
| **手动 SQL 脚本** | 完全控制 | 容易遗漏、版本管理混乱 |

## 决策

**使用 Flyway 管理数据库迁移，SQL 优先策略**：

### 迁移脚本命名规范

```
V{版本号}__{描述}.sql
```

**示例**：
```
V1__Baseline.sql          # 初始 schema
V2__Add_indexes.sql       # 添加索引
V3__Add_foreign_keys.sql  # 添加外键
V4__Add_curation_collections.sql  # 新增场景策展表
```

### 迁移脚本示例（V2__Add_indexes.sql）

```sql
-- 为 items 表添加高频查询索引
CREATE INDEX idx_items_type_status ON items(type, status);
CREATE INDEX idx_items_external_id ON items(external_id);
CREATE INDEX idx_items_slug ON items(slug);

-- 为 reviews 表添加索引
CREATE INDEX idx_reviews_item_id ON reviews(item_id);
CREATE INDEX idx_reviews_user_id ON reviews(user_id);

-- 为 user_items 表添加索引
CREATE INDEX idx_user_items_user_id ON user_items(user_id);
CREATE INDEX idx_user_items_item_id ON user_items(item_id);
```

### Flyway 配置（application.yml）

```yaml
spring:
  flyway:
    enabled: true
    locations: classpath:db/migration
    baseline-on-migrate: true  # 首次迁移时自动创建 baseline
    validate-on-migrate: true  # 迁移前验证脚本是否正确
    clean-disabled: true       # 禁止 clean 命令（防止误删数据）
```

### 数据迁移（Data Import）

**首次启动时自动导入种子数据**：

```java
@Component
public class DataImportService implements CommandLineRunner {

    @Override
    public void run(String... args) throws Exception {
        // 仅在首次启动时导入
        if (itemService.count() == 0) {
            log.info("Importing seed data...");
            importCoffeeData();  // 从 arabica_coffee.csv 导入 130 条咖啡数据
            log.info("Seed data imported successfully");
        }
    }
}
```

**示例：从 CSV 导入咖啡数据**

```java
private void importCoffeeData() throws Exception {
    Resource resource = new ClassPathResource("data/arabica_coffee.csv");
    try (BufferedReader reader = new BufferedReader(new FileReader(resource.getFile()))) {
        String line;
        while ((line = reader.readLine()) != null) {
            String[] fields = line.split(",");

            Item item = new Item();
            item.setType("coffee");
            item.setTitle(fields[0]);
            item.setSlug(slugify(fields[0]));

            Map<String, Object> info = Map.of(
                "origin", fields[1],
                "region", fields[2],
                "variety", fields[3],
                "process", fields[4],
                "altitude", fields[5],
                "cupper_points", Double.parseDouble(fields[6]),
                "total_cup_points", Double.parseDouble(fields[7]),
                "flavor_notes", fields[8]
            );
            item.setInfoJson(objectMapper.writeValueAsString(info));
            item.setStatus(1);  // 已上线

            itemService.save(item);
        }
    }
}
```

### 迁移脚本清单

| 版本 | 脚本 | 说明 |
|------|------|------|
| **V1** | `V1__Baseline.sql` | 创建 9 张核心表 |
| **V2** | `V2__Add_indexes.sql` | 添加 15+ 个索引（加速查询） |
| **V3** | `V3__Add_foreign_keys.sql` | 添加外键 + 级联规则 |
| **V4** | `V4__Add_curation_collections.sql` | 新增场景策展表 |

### V1 Baseline 表结构

```sql
-- items（统一内容表）
CREATE TABLE items (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    type VARCHAR(20) NOT NULL,
    slug VARCHAR(255) NOT NULL UNIQUE,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    cover_url VARCHAR(512),
    info_json JSON,
    status TINYINT DEFAULT 1,
    hot_boost INT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

-- users（用户表）
CREATE TABLE users (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'USER',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- reviews（评论表）
CREATE TABLE reviews (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    item_id BIGINT NOT NULL,
    user_id BIGINT NOT NULL,
    rating TINYINT,
    content TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- tags（标签表）
CREATE TABLE tags (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(50) NOT NULL UNIQUE
);

-- item_tags（物品-标签关联表）
CREATE TABLE item_tags (
    item_id BIGINT NOT NULL,
    tag_id BIGINT NOT NULL,
    PRIMARY KEY (item_id, tag_id)
);

-- user_items（用户-物品关系表）
CREATE TABLE user_items (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    user_id BIGINT NOT NULL,
    item_id BIGINT NOT NULL,
    status VARCHAR(20),
    note TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- scenes（场景表）
CREATE TABLE scenes (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    cover_url VARCHAR(512),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- scene_items（场景-物品关联表）
CREATE TABLE scene_items (
    scene_id BIGINT NOT NULL,
    item_id BIGINT NOT NULL,
    PRIMARY KEY (scene_id, item_id)
);
```

## 替代方案被拒绝的原因

### Liquibase
- ❌ **配置复杂**：支持 XML/YAML/JSON/SQL 多种格式，增加学习成本
- ❌ **过度设计**：DemoNet 的 schema 变更频率不高（每 1-2 周一次），Flyway 足够
- ❌ **社区不如 Flyway 活跃**：遇到问题更容易找到 Flyway 的解决方案

### 手动 SQL 脚本
- ❌ **版本管理混乱**：多人开发时容易遗漏迁移脚本
- ❌ **容易出错**：手动执行 SQL 容易遗漏或执行错误
- ❌ **回滚困难**：手动回滚需要记住每个变更

## 后果

### ✅ 正面影响
- **版本管理清晰**：每个 schema 变更都有对应的迁移脚本
- **自动化迁移**：应用启动时自动执行未执行的迁移
- **团队协作顺畅**：多人开发时 Flyway 自动解决版本冲突
- **SQL 优先**：使用 SQL 而非 DSL，团队无需学习新的语法

### ⚠️ 负面影响
- **不支持自动回滚**：Flyway 的 undo 脚本需要手动编写（Liquibase 支持自动回滚）
- **生产环境风险**：迁移脚本中的 DDL 可能锁表（如 `ALTER TABLE` 大表）
- **数据迁移复杂**：复杂的数据迁移（如字段拆分、数据清洗）需要手动编写 SQL

### 📋 优化方向
1. **Undo 脚本**：为重要的迁移脚本编写 undo 脚本（如 `U4__Rollback_curation_collections.sql`）
2. **零停机迁移**：大表 DDL 使用 `pt-online-schema-change` 或 `gh-ost` 实现零停机
3. **数据迁移脚本**：复杂的数据迁移拆分为独立的迁移脚本（如 `V5__Migrate_data.sql`）
